## 📋 功能概述

AI 主动式助手是一个完整的智能辅助系统，包含 4 个核心功能模块，旨在提升用户的任务管理效率和工作体验。

### 核心价值

- **主动性**: 系统主动检测问题并提醒用户
- **智能化**: AI 驱动的分析和建议
- **完整性**: 覆盖提醒、总结、报告、分解全流程
- **易用性**: 一键操作，无需复杂配置

## 🎯 功能模块

### Phase 1: AI 主动式提醒卡片

**组件**: `AISuggestionCard.vue`  
**触发位置**: 主页面顶部  
**触发时机**: 
- 每日首次打开应用
- 距离上次提醒超过 24 小时
- Snooze 后 1 小时

**功能特性**:
1. **智能检测**:
   - 检测 24 小时内即将逾期的任务
   - 检测高优先级待办任务
   - 检测已完成任务（激励模式）

2. **3种建议类型**:
   - 🔴 **逾期提醒**: "有 X 个任务已逾期，需要立即处理"
   - 🟡 **待办提醒**: "有 X 个高优先级任务待处理"
   - 🟢 **激励消息**: "今天完成了 X 个任务，继续保持！"

3. **3种操作**:
   - **查看详情**: 跳转到对应任务列表
   - **稍后提醒**: 1小时后再次提醒
   - **关闭**: 今日不再提醒

**数据持久化**:
```javascript
localStorage.setItem('ai_suggestion_last_shown', timestamp)
localStorage.setItem('ai_suggestion_snoozed_until', timestamp)
```

### Phase 2: 每日智能总结

**组件**: `DailySummaryModal.vue`  
**触发方式**: 点击顶部 📊 按钮  
**推荐时间**: 每天晚上 8:00 PM

**功能特性**:
1. **4项核心统计**:
   - ✅ 完成任务数
   - ⏰ 逾期任务数
   - 🍅 专注时长（番茄钟）
   - 📈 完成率百分比

2. **今日完成列表**:
   - 显示所有今日完成的任务
   - 标注优先级（高/中/低）
   - 按完成时间排序

3. **AI 智能建议**:
   - 无完成任务 → 鼓励从简单任务开始
   - 有逾期任务 → 建议优先处理逾期
   - 高专注时长（≥4番茄钟）→ 表扬并鼓励保持
   - 多任务完成（≥5个）→ 肯定效率
   - 默认 → 建议设定具体目标

4. **明日计划预览**:
   - 显示前 3 个明日任务
   - 一键跳转到周报生成

**UI 设计**:
- 2×2 统计网格，紫色渐变图标
- AI 建议卡片（左侧边框+渐变背景）
- 滑入动画（slideUp 0.3s）

### Phase 3: AI 周报/月报生成器

**组件**: `AIReportModal.vue` + `aiReportGenerator.js`  
**触发方式**: 
- 从每日总结点击"生成周报"
- 直接访问报告功能

**功能特性**:
1. **报告类型切换**:
   - 周报：最近 7 天数据
   - 月报：最近 30 天数据

2. **完成情况统计**:
   - 总任务数、已完成、进行中、已逾期
   - 完成率百分比
   - 专注时长（番茄钟）
   - 按分类统计（工作/学习/生活）
   - 按优先级统计（高/中/低）

3. **关键工作提取**:
   - 高优先级已完成任务（Top 10）
   - 按分类彩色标签展示
   - 显示完成时间

4. **问题分析**:
   - 逾期任务总数
   - 按分类分组统计
   - AI 智能建议：
     - 逾期>5个 → "逾期任务较多，建议优化时间管理"
     - 工作类逾期>3个 → "工作类任务逾期较多，需要重新评估工作量"

5. **下周/下月计划**:
   - 待办任务统计
   - 高优先级任务列表（Top 5）
   - 3条通用建议

6. **导出功能**:
   - 📋 复制为纯文本
   - 📄 导出PDF（浏览器打印）

**数据结构**:
```javascript
{
  type: 'weekly' | 'monthly',
  period: { start: '2026/02/19', end: '2026/02/26' },
  completionStats: { total, completed, overdue, pending, completionRate, focusHours },
  keyWorks: [{ id, text, category, completedAt }],
  issues: { total, byCategory, suggestions, tasks },
  nextPlan: { total, highPriority, recommendations, tasks }
}
```

### Phase 4: 智能任务分解

**组件**: `SmartTaskSplitter.vue`  
**触发方式**: 点击顶部 🧩 按钮  
**AI 服务**: `aiTaskSplitter.js`

**功能特性**:
1. **AI 驱动分解**:
   - 输入任务标题和描述
   - AI 自动拆解为可执行子任务
   - 智能排序（按执行顺序）

2. **4种任务模板**:
   - **项目开发**: 需求分析 → 技术选型 → 架构设计 → 编码实现 → 测试部署
   - **报告撰写**: 收集资料 → 撰写大纲 → 撰写初稿 → 修改润色 → 最终审核
   - **学习计划**: 制定计划 → 基础学习 → 实践练习 → 总结复习 → 测试评估
   - **活动策划**: 确定主题 → 预算规划 → 场地预订 → 宣传推广 → 现场执行

3. **子任务配置**:
   - 数量可选：3/5/7/10个
   - 时间估算：0.5-8小时（0.5小时步进）
   - 优先级：高/中/低
   - 实时编辑：所有字段可修改

4. **一键创建**:
   - 批量创建所有子任务
   - 自动继承父任务分类
   - 标记父任务已分解

**AI 提示词**:
```
你是一个专业的任务管理助手。请将以下任务拆解为 N 个可执行的子任务。

要求：
1. 每个子任务必须具体、可执行
2. 子任务之间有逻辑顺序
3. 为每个子任务估算所需时间（分钟）
4. 标注优先级（高/中/低）
5. 返回JSON格式
```

**数据结构**:
```javascript
{
  title: '子任务名称',
  description: '详细说明',
  estimatedHours: 2,
  priority: 'high' | 'medium' | 'low'
}
```

## 📊 数据流程

### 提醒卡片流程
```
用户打开应用
  ↓
检查上次提醒时间
  ↓
检查 Snooze 状态
  ↓
分析任务数据
  ↓
生成建议类型
  ↓
显示提醒卡片
  ↓
用户操作（查看/Snooze/关闭）
```

### 报告生成流程
```
用户触发报告
  ↓
选择报告类型（周报/月报）
  ↓
计算时间范围
  ↓
过滤任务数据
  ↓
统计分析
  ↓
生成报告内容
  ↓
渲染展示
  ↓
导出（复制/PDF）
```

### 任务分解流程
```
用户输入任务
  ↓
选择模板/数量
  ↓
调用 AI 服务
  ↓
解析 JSON 结果
  ↓
渲染子任务列表
  ↓
用户编辑
  ↓
一键创建
```

## 🚀 使用指南

### 快速开始

1. **启用提醒卡片**:
   - 打开应用即可自动触发
   - 无需额外配置

2. **查看每日总结**:
   - 点击顶部 📊 按钮
   - 建议每天晚上查看

3. **生成周报**:
   - 从每日总结点击"生成周报"
   - 或直接访问报告功能

4. **分解任务**:
   - 点击顶部 🧩 按钮
   - 输入任务信息
   - 选择模板或AI生成
   - 一键创建子任务

### 最佳实践

1. **提醒卡片**:
   - 每天早上查看提醒
   - 及时处理逾期任务
   - 利用 Snooze 功能合理安排时间

2. **每日总结**:
   - 每天晚上 8:00 PM 查看
   - 根据 AI 建议调整明日计划
   - 定期生成周报

3. **周报/月报**:
   - 每周五生成周报
   - 每月最后一天生成月报
   - 导出PDF存档

4. **任务分解**:
   - 大任务（>4小时）建议分解
   - 优先使用模板
   - AI生成后仔细检查
   - 根据实际情况调整时间估算
