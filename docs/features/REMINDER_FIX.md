# 提醒功能修复说明

## 问题原因

1. **通知配置过于复杂**：使用了不必要的属性（sound、vibrate、ongoing等），这些在Capacitor LocalNotifications中可能不生效
2. **缺少allowWhileIdle**：没有设置允许在设备空闲时触发通知

## 修复内容

### 1. 简化通知配置
现在使用与现有通知（每日摘要、验证码）相同的简单格式：
```javascript
{
  id: task.id,
  title: `🔔 ${task.text}`,
  body: task.description || '任务提醒时间到了！',
  schedule: { 
    at: reminderTime,
    allowWhileIdle: true  // 关键：允许在设备空闲时触发
  }
}
```

### 2. 简化UI
- 移除了"提醒方式"选择（静音/标准/强提醒）
- 只保留"启用提醒"开关和"提醒时间"选择
- Android系统会自动处理通知的声音和震动

### 3. 增强反馈
- 设置提醒成功后显示确认消息
- 控制台输出详细日志便于调试

## 测试步骤

### 方式1：浏览器测试（快速）
```bash
npm run dev
```
1. 打开浏览器（Chrome推荐）
2. 允许通知权限
3. 创建任务，勾选"启用提醒"
4. 设置1分钟后的时间
5. 提交任务
6. 等待1分钟，应该收到浏览器通知

### 方式2：Android真机测试（完整）
```bash
./build-apk.sh
```
1. 安装APK到手机
2. 打开应用，允许通知权限
3. 创建任务，勾选"启用提醒"
4. 设置1分钟后的时间
5. 提交任务
6. **锁屏或切换到其他应用**
7. 等待1分钟，应该收到通知（带声音和震动）

## 通知权限检查

### Android设置路径
设置 → 应用 → TODO App → 通知 → 确保已开启

### 浏览器设置路径
Chrome → 设置 → 隐私和安全 → 网站设置 → 通知 → 允许

## 调试技巧

### 查看控制台日志
创建任务后应该看到：
```
✅ 提醒已设置: 任务名称 时间: Mon Feb 23 2026 21:00:00 GMT+0800
```

### 手动测试通知
在浏览器控制台执行：
```javascript
import { LocalNotifications } from '@capacitor/local-notifications'
await LocalNotifications.schedule({
  notifications: [{
    id: 88888,
    title: '测试通知',
    body: '这是一个测试',
    schedule: { 
      at: new Date(Date.now() + 10000),  // 10秒后
      allowWhileIdle: true
    }
  }]
})
```

## 常见问题

### Q: 还是没有收到通知？
A: 检查以下几点：
1. 通知权限是否已授予
2. 提醒时间是否是未来时间
3. Android系统是否开启了省电模式（可能延迟通知）
4. 查看控制台是否有错误日志

### Q: 通知没有声音？
A: 
- Android：检查系统通知设置，确保通知声音已开启
- 浏览器：浏览器通知通常没有声音，这是正常的

### Q: 锁屏后没有通知？
A: 
- 确保设置了 `allowWhileIdle: true`（已修复）
- 检查Android省电模式设置
- 某些手机厂商（小米、华为等）可能需要额外设置"后台运行"权限
