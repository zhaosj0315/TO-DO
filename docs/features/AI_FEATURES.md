## 1. AI模型配置

### 功能概述

统一的AI模型配置中心，支持多种AI服务提供商。

### 支持的模型

- **盘古AI** - 华为盘古大模型
- **OpenAI** - GPT系列模型
- **Ollama** - 本地部署模型
- **自定义** - 任何兼容OpenAI API的模型

### 配置方法

1. 进入个人主页 → AI配置
2. 添加模型：
   - 模型名称
   - API地址
   - API密钥
   - 厂商类型
3. 设置默认模型
4. 测试连接

### 配置导入导出

- **导出配置**: 保存为JSON文件
- **导入配置**: 从JSON文件恢复

### 使用场景

所有AI功能共享此配置：
- AI问答
- 任务拆分
- 智能建议
- 文本增强

## 2. AI智能问答

### 功能概述

自然语言问答系统，可查询任务数据、获取建议。

### 核心功能

**数据上下文**：
- 所有任务数据
- 执行日志
- 番茄钟历史
- 统计信息

**快捷问题**（6个）：
1. 今天完成了哪些任务？
2. 本周任务情况如何？
3. 有哪些高优先级待办？
4. 最近遇到了什么阻碍？
5. 给我一些效率建议
6. 分析我的工作习惯

**智能分析**：
- 时间维度查询（今天/本周/本月）
- 状态查询（待办/已完成/已逾期）
- 执行洞察（阻碍/解决方案/进度）

### 使用方法

1. 点击🤖 AI问答按钮
2. 选择快捷问题或输入自定义问题
3. 查看AI回答
4. 继续对话

### 对话历史

- 多对话管理（类似ChatGPT）
- 时间智能分组（今天/昨天/最近7天/更早）
- 对话切换和删除

## 3. AI任务拆分

### 功能概述

将复杂任务自动拆分为可执行的子任务。

### 核心功能

**智能拆解**：
- 输入任务标题和描述
- AI自动生成子任务列表
- 预估每个子任务时长（0.5-8小时）

**4种拆分模板**：
1. **快速拆分** - 3-5个子任务
2. **详细拆分** - 5-8个子任务
3. **时间优先** - 按时间顺序拆分
4. **优先级优先** - 按重要性拆分

**子任务预览**：
- Bottom Sheet展示拆分结果
- 支持编辑和删除
- 一键创建所有子任务

**依赖关系**：
- 子任务自动依赖父任务
- 完成父任务后子任务才能开始

### 使用方法

**方式1：任务详情页**
1. 打开任务详情
2. 点击"🤖 AI拆分"按钮
3. 选择拆分模板
4. 预览并编辑子任务
5. 确认创建

**方式2：任务预览**
1. 创建新任务时点击"预览"
2. 在预览弹窗中使用AI拆分
3. 父任务和子任务一次性保存

主动检测任务状态，提供智能提醒和建议。

### 核心功能

**每日智能总结**：
- 今日完成统计

**报告生成器**：
- 日报
- 周报
- 月报
- 季报
- 半年报
- 年报
- 自定义日期范围

**报告内容**（11章节）：
1. 智能总结
2. 数据概览
3. 完成任务明细
4. 已完成情况
5. 本期目标
6. 本期进展
7. 本周进展
8. 关键工作
9. 风险与问题

**报告导出**：
- HTML格式（可打印为PDF）

### 使用方法

1. 点击🤖 AI助手按钮
2. 查看智能提醒卡片

## 5. AI文本选择菜单

### 功能概述

选中文本后弹出AI菜单，提供智能操作。

### 支持的操作

1. **AI续写** - 根据上下文续写内容
2. **AI优化** - 优化文本表达
3. **AI总结** - 提取关键信息

### 使用场景

- 任务描述编辑
- 日志内容编辑
- 任务标题编辑

### 使用方法

1. 选中文本（长按或拖动）
2. 弹出AI菜单
3. 选择操作
4. 查看结果
5. 采纳或取消

### 菜单位置

- 任务详情页 - 描述字段
- 添加日志弹窗 - 内容字段
- 全屏编辑器 - 标题和描述字段

## 🔧 技术实现

### 统一API接口

所有AI功能使用统一的API接口：
```
POST /v1/chat/completions
```

### 数据格式

```json
{
  "model": "model-name",
  "messages": [
    {"role": "system", "content": "系统提示"},
    {"role": "user", "content": "用户输入"}
  ],
  "stream": true
}
```

### 流式响应

支持流式响应（SSE），实时显示AI生成内容。

### 错误处理

- 自动降级（流式 → 非流式）
- 超时重试
- 错误提示

## ⚠️ 注意事项

### 网络要求

- 需要稳定的网络连接
- 本地模型（Ollama）除外

### API密钥安全

- 密钥存储在本地
- 不会上传到服务器
- 建议定期更换

### 数据隐私

- 任务数据仅在AI调用时发送
- 不会永久存储在AI服务器
- 建议使用本地模型处理敏感数据

### 成本控制

- OpenAI等服务按调用次数收费
- 建议设置每月预算
- 优先使用本地模型

## 📝 常见问题

### Q: AI响应很慢怎么办？

A: 
1. 检查网络连接
2. 切换到更快的模型
3. 使用本地模型（Ollama）

### Q: AI生成的内容不准确？

A:
1. 优化Prompt描述
2. 提供更多上下文
3. 切换到更智能的模型

### Q: 如何使用本地模型？

A:
1. 安装Ollama
2. 下载模型（如llama2）
3. 在AI配置中添加本地模型
4. API地址：http://localhost:11434

### Q: 可以同时使用多个模型吗？

A:
可以。在AI配置中添加多个模型，根据场景切换使用。
