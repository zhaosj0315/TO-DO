# 看板+模板+日历 实现方案

## 🎯 实施计划（v0.10.0）

### Phase 1: 模板系统（1-2天）✅ 优先
**原因**：最简单，立即可用，为其他功能打基础

### Phase 2: 看板视图（2-3天）
**原因**：用户价值最高，视觉冲击力强

### Phase 3: 日历视图（2-3天）
**原因**：时间维度管理，完善多视图体系

## 📋 Phase 1: 模板系统

### 数据结构
```javascript
// stores/templateStore.js
{
  id: 'weekly-report',
  name: '工作周报',
  icon: '📊',
  category: 'work',
  content: {
    title: '{{week}} 工作周报',
    description: `## 本周完成\n- \n\n## 下周计划\n- \n\n## 遇到问题\n- `,
    category: 'work',
    priority: 'medium',
    tags: ['#周报']
  },
  variables: {
    week: () => `第${getWeekNumber()}周`,
    date: () => new Date().toLocaleDateString('zh-CN'),
    time: () => new Date().toLocaleTimeString('zh-CN', {hour: '2-digit', minute: '2-digit'}),
    user: () => currentUser.value
  }
}
```

## 📊 Phase 2: 看板视图

### 数据结构（复用现有任务）
```javascript
// 不需要新字段，使用现有 status
status: 'pending' → 待办列
status: 'in_progress' → 进行中列（新增状态）
status: 'completed' → 已完成列
```

### UI布局
```
┌─────────────────────────────────────────┐
│ [列表] [看板] [日历] [甘特图] [关系图] │
├─────────────────────────────────────────┤
│  待办 (5)    │  进行中 (2)  │  已完成 (8) │
│ ┌─────────┐  │ ┌─────────┐ │ ┌─────────┐│
│ │ 任务1   │  │ │ 任务6   │ │ │ 任务9   ││
│ │ 高优先级│  │ │ 中优先级│ │ │ 低优先级││
│ └─────────┘  │ └─────────┘ │ └─────────┘│
│ ┌─────────┐  │ ┌─────────┐ │ ┌─────────┐│
│ │ 任务2   │  │ │ 任务7   │ │ │ 任务10  ││
│ └─────────┘  │ └─────────┘ │ └─────────┘│
└─────────────────────────────────────────┘
```

### 技术方案
- 使用 `vuedraggable` 库（支持触摸事件）
- **移动端优化**：
  - 长按触发拖拽（500ms）
  - 触觉反馈（震动）
  - 拖拽时卡片放大 1.05 倍
  - 半透明拖拽预览
- **备选方案**：点击卡片弹出状态选择器
  ```
  点击卡片 → 弹窗：
  ┌─────────────┐
  │ 移动到：    │
  │ ○ 待办      │
  │ ● 进行中    │
  │ ○ 已完成    │
  └─────────────┘
  ```
- 列内排序
- 响应式3列布局（移动端可横向滚动）

### 实现文件
- `src/components/KanbanView.vue` - 看板主组件
- `src/components/KanbanColumn.vue` - 看板列
- `src/components/KanbanCard.vue` - 看板卡片（复用TaskCard）

## 🎨 UI设计原则

1. **视图平等**：所有视图地位平等，无主次之分
2. **数据共享**：所有视图操作同一份数据
3. **状态同步**：切换视图时保持筛选状态
4. **移动优先**：所有视图适配移动端
5. **渐进增强**：不破坏现有功能
