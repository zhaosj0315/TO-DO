## 📋 功能概述

AI模型配置功能允许用户配置和管理多个AI模型，支持本地Ollama、OpenAI兼容API和自定义API。v1.7.7版本进行了重大优化，增强了兼容性和易用性。

### 核心特性
- **多模型管理**: 支持添加、编辑、删除多个AI模型
- **厂商配置复用**: 保存厂商配置，快速添加同厂商的其他模型
- **手动输入支持**: 当厂商不提供模型列表接口时，支持手动输入模型名称
- **流式响应降级**: 自动检测并降级为非流式响应
- **连接测试**: 测试模型连接状态
- **配置导入导出**: 备份和恢复模型配置

## 🎯 v1.7.7 优化内容

### 1. 手动输入模型支持 ✨
**问题**: 部分API服务商不提供 `/v1/models` 接口，无法自动获取模型列表。

**解决方案**:
- 当"获取模型"失败（404错误）时，自动显示手动输入框
- 用户可直接输入模型名称（如 `gpt-4o-mini`）
- 输入后可测试连接验证模型名称是否正确

**使用场景**:
- 第三方API代理服务
- 自建API服务
- 不支持模型列表接口的服务商

### 2. 自动创建厂商配置 ✨
**问题**: 手动输入模型时，厂商配置丢失，导致添加失败。

**解决方案**:
- 手动输入模型时，自动创建并保存厂商配置
- 厂商配置包含：类型、URL、API Key
- 下次添加同厂商模型时可直接复用

**优势**:
- 无需重复输入地址和API Key
- 支持快速添加同厂商的多个模型
- 配置持久化，刷新页面不丢失

### 3. 流式响应自动降级 ✨
**问题**: 部分服务商不支持流式响应（`stream: true`），导致AI问答无法使用。

**解决方案**:
- AI问答优先尝试流式响应
- 如果返回400错误，自动切换为非流式响应
- 非流式响应一次性返回完整内容

**兼容性**:
- ✅ 支持流式的厂商：打字机效果
- ✅ 不支持流式的厂商：一次性返回

### 4. 流式数据解析优化 ✨
**问题**: 不同厂商的流式数据格式不同，导致解析失败。

**解决方案**:
- 优化数据分割逻辑：按 `\n\ndata:` 分割
- 处理不完整的JSON数据块
- 兼容多种流式数据格式

**支持格式**:
- OpenAI标准格式
- 第三方代理格式
- 自定义格式

### 5. API路径统一修复 🐛
**问题**: 多个AI服务使用错误的API路径（缺少 `/v1/`）。

**修复内容**:
- 统一所有AI服务使用 `/v1/chat/completions`
- 修复9个AI服务文件的API路径
- 自动规范化URL（移除末尾斜杠和已有路径）

**修复文件**:
- aiTaskSplitter.js
- aiDailyPlanner.js
- aiChatService.js
- aiTextEnhancer.js
- aiTextService.js
- aiClassifier.js
- aiTaskExtractor.js
- aiTaskGenerator.js
- AIChat.vue

### 6. 按钮文字优化 🎨
**问题**: 导出/导入按钮只有图标（📤/📥），容易混淆。

**解决方案**:
- 添加文字标签："📤 导出" / "📥 导入"
- 测试按钮："🔍 测试全部"
- 提升可用性和可读性

## 🚀 使用方法

### 1. 添加模型（自动获取）

#### 步骤
1. 打开"AI模型配置"页面
2. 选择模型类型（本地Ollama / OpenAI / 自定义）
3. 输入厂商地址（如 `https://api.openai.com`）
4. 输入API Key（如需要）
5. 点击"🔄 获取模型"按钮
6. 从下拉框选择模型
7. 点击"🔍 测试连接"验证
8. 点击"➕ 添加模型"保存

#### 示例：添加OpenAI模型
```
类型: OpenAI
地址: https://api.openai.com
API Key: sk-xxxxxxxxxxxxx
模型: gpt-4o-mini
```

### 2. 添加模型（手动输入）

#### 步骤
1. 打开"AI模型配置"页面
2. 选择模型类型
3. 输入厂商地址
4. 输入API Key
5. 点击"🔄 获取模型"（会显示404错误）
6. **在出现的手动输入框中输入模型名称**
7. 点击"🔍 测试连接"验证
8. 点击"➕ 添加模型"保存

#### 示例：添加第三方代理模型
```
类型: OpenAI
地址: https://api.longcat.chat/openai
API Key: ak_xxxxxxxxxxxxx
模型名称: LongCat-Flash-Chat (手动输入)
```

### 3. 使用已保存的厂商配置

#### 步骤
1. 点击"📦 快速选择已保存的配置..."下拉框
2. 选择已保存的厂商配置
3. 系统自动填充地址和API Key
4. 点击"🔄 获取模型"或手动输入模型名称
5. 点击"➕ 添加模型"保存

#### 优势
- 无需重复输入地址和API Key
- 快速添加同厂商的多个模型
- 配置持久化

### 4. 测试模型连接

#### 单个模型测试
1. 在已配置模型列表中找到目标模型
2. 点击"🔍"按钮
3. 等待测试结果
4. 查看状态指示器：🟢在线 / 🔴离线

#### 批量测试
1. 点击"🔍 测试全部"按钮
2. 系统依次测试所有模型
3. 查看每个模型的状态

### 5. 设置默认模型

#### 步骤
1. 在"⭐ 默认使用模型"区域
2. 从下拉框选择默认模型
3. 系统自动保存
4. 所有AI功能将使用该模型

### 6. 导出/导入配置

#### 导出配置
1. 点击"📤 导出"按钮
2. 系统生成JSON文件
3. 保存到本地

#### 导入配置
1. 点击"📥 导入"按钮
2. 选择JSON文件
3. 系统自动导入所有模型配置

## 💻 技术实现

### 数据结构

#### 模型配置
```javascript
{
  id: String,              // 模型ID（时间戳）
  type: String,            // 类型: 'local' | 'openai' | 'custom'
  name: String,            // 显示名称（自动生成：厂商 - 模型名）
  url: String,             // 厂商基础URL
  apiKey: String,          // API Key（可选）
  modelName: String,       // 模型名称
  providerId: String,      // 关联的厂商配置ID
  status: String,          // 状态: 'online' | 'offline' | 'testing' | 'unknown'
  stats: Object            // 统计信息（调用次数、成功率）
}
```

#### 厂商配置
```javascript
{
  id: String,              // 厂商配置ID（时间戳）
  type: String,            // 类型: 'local' | 'openai' | 'custom'
  url: String,             // 基础URL（规范化后）
  apiKey: String,          // API Key
  createdAt: String        // 创建时间（ISO格式）
}
```

### 核心方法

#### 1. fetchAvailableModels()
获取可用模型列表。

**逻辑**:
1. 构造API URL（`/v1/models` 或 `/api/tags`）
2. 发送GET请求
3. 解析响应数据
4. 成功：显示模型下拉框
5. 失败：显示手动输入框

#### 2. addModel()
添加模型配置。

**逻辑**:
1. 验证必填字段
2. 获取或创建厂商配置
3. 创建模型对象
4. 保存到localStorage
5. 如果是第一个模型，设为默认

#### 3. testConnection()
测试模型连接。

**逻辑**:
1. 构造测试请求（发送"Hello"消息）
2. 优先尝试流式响应
3. 如果400错误，降级为非流式
4. 返回测试结果

#### 4. callOpenAIStream()
OpenAI流式调用（带降级）。

**逻辑**:
1. 尝试流式响应（`stream: true`）
2. 如果400错误，切换为非流式（`stream: false`）
3. 非流式：一次性返回完整内容
4. 流式：逐字显示内容

## 🐛 故障排除

### 问题1：获取模型列表失败（404）
**原因**: 服务商不提供 `/v1/models` 接口

**解决方案**: 使用手动输入功能

### 问题2：测试连接失败（400）
**原因**: 模型名称错误或不支持

**解决方案**: 
1. 检查模型名称拼写
2. 查看服务商文档确认支持的模型
3. 尝试其他常见模型名称

### 问题3：AI问答无响应
**原因**: 流式响应不支持

**解决方案**: 系统会自动降级，如果仍无响应，检查：
1. API Key是否正确
2. 网络连接是否正常
3. 服务商是否正常运行

### 问题4：厂商配置丢失
**原因**: 旧版本bug（v1.7.7已修复）

**解决方案**: 升级到v1.7.7，系统会自动创建厂商配置
