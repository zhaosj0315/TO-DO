## 📋 功能概述

任务依赖关系系统是v1.7.7新增的核心功能，允许用户设置任务之间的依赖关系。当一个任务需要等待另一个任务完成后才能开始时，可以通过依赖关系进行管理。

### 设计理念
- **极简设计**: 1个字段、5个方法、3个UI组件
- **单向依赖**: 一个任务只能等待一个其他任务
- **智能通知**: 自动通知等待的任务可以开始
- **自动清理**: 删除任务时自动清除相关依赖

---

## 🎯 使用场景

### 场景1：简单依赖
```
任务A → 任务B
```
任务B等待任务A完成后才能开始。

**示例**:
- A: 完成需求文档
- B: 开始编码（等待A完成）

### 场景2：依赖链
```
任务A → 任务B → 任务C → 任务D
```
形成一条依赖链，按顺序完成。

**示例**:
- A: 设计原型
- B: 前端开发（等待A）
- C: 后端开发（等待B）
- D: 测试（等待C）

### 场景3：多任务等待同一任务
```
任务A → 任务B
任务A → 任务C
任务A → 任务D
```
多个任务都等待同一个任务完成。

**示例**:
- A: 完成API文档
- B: 前端对接（等待A）
- C: 移动端对接（等待A）
- D: 测试用例编写（等待A）

## 🚀 使用方法

### 1. 设置依赖关系

#### 方法一：任务详情页设置
1. 打开任务详情页（点击任务卡片）
2. 找到"依赖关系"区域
3. 点击"设置等待任务"按钮
4. 在弹出的选择器中搜索或筛选任务
5. 选择要等待的任务
6. 点击"确定"保存

#### 方法二：AI任务拆分自动设置
1. 使用AI任务拆分功能
2. 系统自动为子任务设置依赖关系
3. 所有子任务自动等待父任务完成

### 2. 查看依赖状态

#### 任务卡片徽章
- 🔒 **等待中**（黄色徽章）: 该任务正在等待其他任务完成
- 🔓 **被依赖**（蓝色徽章）: 有其他任务正在等待该任务
- ✅ **无依赖**: 没有依赖关系

#### 任务详情页
- 显示"等待任务"名称（可点击跳转）
- 显示"被依赖任务"列表
- 显示依赖状态图标

### 3. 清除依赖关系

#### 手动清除
1. 打开任务详情页
2. 点击"清除依赖"按钮
3. 确认清除

#### 自动清除
- 删除任务时，系统自动清除所有相关依赖关系
- 完成被等待的任务时，系统自动通知等待的任务

## 💻 技术实现

### 数据结构

#### 任务对象扩展
```javascript
{
  id: String,              // 任务ID
  title: String,           // 任务标题
  waitFor: String | null,  // 等待的任务ID（新增字段）
  // ... 其他字段
}
```

### 核心方法

#### 1. setTaskDependency(taskId, waitForTaskId)
设置任务依赖关系。

**参数**:
- `taskId`: 当前任务ID
- `waitForTaskId`: 要等待的任务ID

**逻辑**:
1. 验证两个任务都存在
2. 防止循环依赖
3. 更新任务的waitFor字段
4. 保存到localStorage

#### 2. clearTaskDependency(taskId)
清除任务依赖关系。

**参数**:
- `taskId`: 任务ID

**逻辑**:
1. 将任务的waitFor字段设为null
2. 保存到localStorage

#### 3. getWaitingTasks(taskId)
获取等待指定任务的所有任务。

**参数**:
- `taskId`: 任务ID

**返回**:
- 等待该任务的任务列表

#### 4. notifyWaitingTasks(taskId)
通知等待指定任务的所有任务。

**参数**:
- `taskId`: 已完成的任务ID

**逻辑**:
1. 查找所有等待该任务的任务
2. 发送本地通知
3. 通知内容："{任务名称} 已完成，您可以开始 {等待任务名称} 了"

#### 5. cleanupDependencies(taskId)
清理任务的所有依赖关系。

**参数**:
- `taskId`: 被删除的任务ID

**逻辑**:
1. 清除该任务的waitFor字段
2. 清除所有等待该任务的任务的waitFor字段

### UI组件

#### 1. WaitForSelector.vue
等待任务选择器组件。

**功能**:
- 搜索任务
- 筛选任务（按状态、分类、优先级）
- 单选模式
- 防止选择自己
- 防止循环依赖

**Props**:
- `visible`: 是否显示
- `currentTaskId`: 当前任务ID
- `excludeTaskIds`: 排除的任务ID列表

**Events**:
- `select`: 选择任务
- `close`: 关闭选择器

#### 2. DependencyManager.vue
依赖关系管理组件。

**功能**:
- 显示当前依赖状态
- 设置依赖关系
- 清除依赖关系
- 跳转到依赖任务

**Props**:
- `task`: 任务对象

**Events**:
- `update`: 依赖关系更新

#### 3. TaskDetailModal.vue (扩展)
任务详情页扩展。

**新增功能**:
- 依赖关系区域
- 等待任务显示
- 被依赖任务列表
- 依赖状态徽章

## 🐛 已知问题

### 问题1：不支持多重依赖
**描述**: 一个任务只能等待一个其他任务，不支持同时等待多个任务。

**影响**: 复杂依赖场景受限

**计划**: v1.8.0考虑支持

### 问题2：不支持依赖链可视化
**描述**: 无法以图形方式查看完整的依赖链。

**影响**: 复杂依赖关系难以理解

**计划**: v1.8.0考虑添加依赖关系图

## 🔄 数据迁移

### 自动迁移
系统启动时自动为所有旧任务添加 `waitFor: null` 字段。

**迁移代码**:
```javascript
// offlineTaskStore.js
const migrateTaskDependencies = () => {
  tasks.value.forEach(task => {
    if (!task.hasOwnProperty('waitFor')) {
      task.waitFor = null
    }
  })
  saveTasks()
}
```

### 兼容性
- ✅ 向后兼容：旧版本数据自动迁移
- ✅ 向前兼容：新字段不影响旧功能
