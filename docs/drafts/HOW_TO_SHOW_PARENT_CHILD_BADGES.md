# 如何让任务显示父子关系徽章

## ❓ 问题

任务卡片上只显示了 `⬆️ 依赖×N` 和 `⬇️ 被依赖×N`，但是没有显示 `👨‍👦 父任务×1` 和 `👶 子任务×N`。

## ✅ 原因

你的任务是通过**手动添加依赖**创建的，所以只有依赖关系，没有父子关系。

## 🔧 解决方案

### 方法1：使用AI任务拆分（推荐）

1. **打开任务详情**
   - 点击任务标题或描述

2. **找到AI任务拆分按钮**
   - 在任务详情页底部
   - 图标：🤖 或 "AI拆分"

3. **输入拆分信息**
   - 任务标题：例如 "开发新功能"
   - 任务描述：详细描述任务内容
   - 选择拆分模板：快速拆分/详细拆分/时间优先/优先级优先

4. **预览并创建子任务**
   - AI会自动生成3-5个子任务
   - 可以编辑子任务标题、描述、优先级
   - 点击"创建全部子任务"

5. **查看结果**
   - 父任务显示：`👶 子任务×N`
   - 子任务显示：`👨‍👦 父任务×1`

### 方法2：理解当前任务关系

你当前的任务关系：

```
浏览并挑选30分钟口语练习课程
  ⬇️ 被依赖×5  ← 有5个任务依赖它

绘制系统架构图
  ⬆️ 依赖×1    ← 依赖1个前置任务

收集API接口信息
  ⬆️ 依赖×1    ← 依赖1个前置任务
```

这些都是**依赖关系**（手动添加），不是**父子关系**（AI拆分）。

## 📊 两种关系对比

| 特征 | 父子关系 | 依赖关系 |
|------|---------|---------|
| 创建方式 | AI任务拆分 | 手动添加前置任务 |
| 徽章显示 | 👨‍👦 父任务×1 / 👶 子任务×N | ⬆️ 依赖×N / ⬇️ 被依赖×N |
| 用途 | 组织结构、层级关系 | 执行顺序、依赖约束 |
| 是否强制顺序 | ❌ 否 | ✅ 是 |

## 🎯 实际案例

### 案例1：项目拆解（父子关系）

```
项目：开发新功能
  👶 子任务×4
    ├─ 子任务1：需求分析 (👨‍👦 父任务×1)
    ├─ 子任务2：设计方案 (👨‍👦 父任务×1)
    ├─ 子任务3：编码实现 (👨‍👦 父任务×1)
    └─ 子任务4：测试验收 (👨‍👦 父任务×1)
```

### 案例2：任务依赖（依赖关系）

```
任务A：准备材料
  ⬇️ 被依赖×1

任务B：开始制作
  ⬆️ 依赖×1  ← 依赖任务A
```

### 案例3：混合使用

```
项目：写论文
  👶 子任务×3
    ├─ 子任务1：文献调研
    │   👨‍👦 父任务×1
    │   ⬆️ 依赖×1  ← 依赖外部任务
    │
    ├─ 子任务2：撰写初稿
    │   👨‍👦 父任务×1
    │   ⬆️ 依赖×1  ← 依赖子任务1
    │
    └─ 子任务3：修改定稿
        👨‍👦 父任务×1
        ⬆️ 依赖×1  ← 依赖子任务2
```

## 💡 建议

### 什么时候用AI拆分？

- ✅ 大型项目需要分解为多个步骤
- ✅ 任务复杂，需要结构化管理
- ✅ 想要快速生成子任务列表

### 什么时候用手动依赖？

- ✅ 任务之间有明确的先后顺序
- ✅ 需要强制执行顺序（完成A才能开始B）
- ✅ 跨项目的任务依赖

### 可以同时使用吗？

✅ **可以！** 子任务也可以手动添加依赖：

```
子任务2：设计方案
  👨‍👦 父任务×1  ← AI拆分
  ⬆️ 依赖×1      ← 手动添加
```

## 🔍 验证方法

### 检查任务是否有父子关系

1. 打开浏览器开发者工具（F12）
2. 在Console中运行：

```javascript
// 查看所有任务的父子关系
JSON.parse(localStorage.getItem('tasks_你的用户名')).forEach(task => {
  if (task.parentTaskId) {
    console.log(`子任务: ${task.text} → 父任务ID: ${task.parentTaskId}`)
  }
  const children = JSON.parse(localStorage.getItem('tasks_你的用户名'))
    .filter(t => t.parentTaskId === task.id)
  if (children.length > 0) {
    console.log(`父任务: ${task.text} → 子任务数: ${children.length}`)
  }
})
```

## 📞 需要帮助？

如果你想把现有的依赖关系转换为父子关系：

1. 创建一个新的父任务
2. 使用AI拆分生成子任务
3. 删除旧的依赖关系
4. 重新建立依赖关系（如果需要）
