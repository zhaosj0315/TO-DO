## 🎯 使用流程

### 添加新模型（完整流程）

1. **选择模型类型**
   - 本地Ollama
   - OpenAI
   - 自定义

2. **输入厂商地址**
   - Ollama: `http://192.168.31.159:11434`
   - OpenAI: `https://api.openai.com`
   - 自定义: 任何兼容API

3. **输入API Key**（如果需要）
   - OpenAI 和自定义类型需要

4. **点击🔄刷新模型**
   - 自动获取可用模型列表
   - 填充到"选择模型"下拉框

5. **选择模型**
   - 从下拉框选择模型
   - 自动生成显示名称（如"Ollama - gemma2:2b"）
   - 可自定义修改名称

6. **测试连接**
   - 点击"🔍 测试连接"按钮
   - 等待测试结果
   - ✅ 连接成功 或 ❌ 连接失败

7. **添加模型**
   - 测试成功后点击"➕ 添加模型"
   - 模型添加到列表
   - 可在"默认使用模型"中选择

## 🔧 技术实现

### 自动生成模型名称

```javascript
watch(() => newModel.value.modelName, (modelName) => {
  if (modelName) {
    const typeLabel = {
      'local': 'Ollama',
      'openai': 'OpenAI',
      'custom': '自定义'
    }[newModel.value.type] || '未知'
    
    newModel.value.name = `${typeLabel} - ${modelName}`
  }
})
```

## 💡 使用建议

### 最佳实践

1. **先测试再添加**
   - 每次添加模型前先测试连接
   - 确保模型可用后再添加

2. **使用自动生成的名称**
   - 默认格式清晰直观
   - 除非有特殊需求，不建议修改

3. **定期测试已有模型**
   - 在模型列表中添加"测试"按钮（后续优化）
   - 确保已配置模型仍然可用

### 常见问题

**Q: 刷新模型失败怎么办？**
A: 检查地址是否正确，Ollama是否运行，API Key是否有效

**Q: 测试连接失败怎么办？**
A: 查看错误提示，检查网络连接，验证模型名称是否正确

**Q: 可以修改显示名称吗？**
A: 可以，选择模型后会显示名称输入框，支持自定义
