# 提醒功能测试指南

## 快速测试步骤

### 1. 启动应用
```bash
npm run dev
# 或者构建APK测试
./build-apk.sh
```

### 2. 创建带提醒的任务

1. 登录应用
2. 点击"展开"按钮（↓图标）
3. 输入任务名称，例如："测试提醒功能"
4. 向下滚动，找到"🔔 启用提醒"区域
5. 勾选"启用提醒"复选框
6. 选择提醒时间（建议设置为1-2分钟后）
7. 选择提醒方式：
   - **静音**：只显示通知，无声音
   - **标准**：通知 + 声音 + 震动
   - **强提醒**：持续通知 + 声音 + 震动
8. 点击 ✓ 提交

### 3. 验证提醒设置

- 任务创建后，在任务列表中应该看到 🔔 图标
- 鼠标悬停在 🔔 图标上，应该显示提醒时间

### 4. 等待提醒触发

- 等待到设置的提醒时间
- 应该收到系统通知
- 根据选择的提醒方式，通知会有不同的表现：
  - 静音：只有通知弹窗
  - 标准：通知 + 默认铃声 + 震动
  - 强提醒：持续通知（不可滑动关闭）+ 铃声 + 震动

### 5. 测试取消提醒

#### 方式1：完成任务
1. 勾选任务的复选框
2. 任务标记为完成
3. 提醒应该被自动取消

#### 方式2：删除任务
1. 点击任务的 🗑️ 按钮
2. 任务被删除
3. 提醒应该被自动取消

### 6. 边界测试

#### 测试过去时间
1. 尝试设置过去的时间作为提醒时间
2. 提交任务
3. 应该看到提示："提醒时间已过，未设置提醒"

#### 测试不启用提醒
1. 创建任务但不勾选"启用提醒"
2. 任务列表中不应该显示 🔔 图标

### ❌ 常见问题

#### 问题1：没有收到通知
**可能原因**：
- 未授予通知权限
- 系统通知被禁用
- 提醒时间设置错误

**解决方法**：
- 检查应用通知权限（设置 → 应用 → TODO App → 通知）
- 确认提醒时间是未来时间
- 查看浏览器控制台是否有错误

#### 问题2：强提醒无法关闭
**说明**：这是预期行为，强提醒模式使用 `ongoing: true`，需要用户手动处理任务或在通知中心清除。

#### 问题3：提醒图标不显示
**检查**：
- 确认已勾选"启用提醒"
- 确认已选择提醒时间
- 刷新页面重新加载任务

## 调试技巧

### 查看控制台日志
```javascript
// 在浏览器控制台查看
console.log('提醒已设置:', task.text, '时间:', reminderTime)
```

### 检查本地存储
```javascript
// 在浏览器控制台执行
import { Preferences } from '@capacitor/preferences'
const { value } = await Preferences.get({ key: 'tasks_用户名' })
console.log(JSON.parse(value))
```

### 手动触发通知测试
```javascript
// 在浏览器控制台执行
import { LocalNotifications } from '@capacitor/local-notifications'
await LocalNotifications.schedule({
  notifications: [{
    id: 99999,
    title: '测试通知',
    body: '这是一个测试通知',
    schedule: { at: new Date(Date.now() + 5000) } // 5秒后
  }]
})
```
