# 演示模式改进方案

## 🔍 当前问题分析

### 1. **步骤覆盖不全面**
- ❌ 缺少任务编辑功能的介绍
- ❌ 缺少任务详情页（Bottom Sheet）的介绍
- ❌ 缺少执行日志系统的介绍（v1.7.0 核心功能）
- ❌ 缺少数据导入导出的介绍
- ❌ 缺少重复任务的介绍
- ❌ 缺少搜索功能的介绍
- ❌ 缺少刷新按钮的介绍

### 2. **描述不够详细**
- ❌ 只说"点击按钮"，没说明具体效果
- ❌ 没有说明操作的意义和使用场景
- ❌ 缺少实际操作示例

### 3. **交互体验不足**
- ❌ 没有引导用户实际操作
- ❌ 只能被动阅读，无法互动
- ❌ 缺少操作反馈

### 4. **步骤顺序不合理**
- ❌ 先介绍任务卡片，但用户可能还没创建任务
- ❌ 番茄钟、置顶等功能需要先有任务才能演示
- ❌ 缺少"创建示例任务"的步骤

### 5. **缺少关键功能**
- ❌ 没有介绍任务执行日志（v1.7.0 最重要的功能）
- ❌ 没有介绍任务详情页的时间轴
- ❌ 没有介绍进度追踪
- ❌ 没有介绍标签系统

### Phase 2: 优化描述（立即实施）

#### 描述模板
```
标题: [功能名称]
描述: 
1. 这是什么？[功能定义]
2. 怎么用？[操作步骤]
3. 有什么用？[使用场景]
4. 小提示：[注意事项]
```

#### 示例对比

**优化前**:
```
标题: 任务卡片
描述: 点击任务标题或描述可以查看详情、添加执行日志。
      点击左侧的复选框可以标记任务完成。
```

**优化后**:
```
标题: 任务卡片操作
描述: 
• 点击标题/描述 → 打开任务详情页（Bottom Sheet）
• 点击复选框 ☑️ → 标记完成/取消完成
• 点击 🍅 → 开始番茄钟计时
• 点击 📌 → 置顶/取消置顶
• 点击 🗑️ → 删除任务（移入回收站）

💡 小提示：任务卡片上的徽章显示日志数量（💬）和进度（📊）
```

### Phase 3: 互动式教程（后续实施）

#### 3.1 引导用户实际操作
```javascript
// 步骤 6: 创建示例任务
{
  target: '.add-form-row-main input',
  icon: '✍️',
  title: '现在，创建您的第一个任务！',
  description: '请在输入框中输入"学习使用 TO-DO App"，然后按回车键。',
  interactive: true,  // 🆕 互动模式
  waitForAction: 'task-created',  // 🆕 等待用户操作
  nextButtonDisabled: true  // 🆕 禁用"下一步"按钮
}
```

#### 3.2 操作验证
```javascript
// 监听用户操作
watch(() => taskStore.tasks.length, (newLength, oldLength) => {
  if (currentStep.waitForAction === 'task-created' && newLength > oldLength) {
    // 用户成功创建任务
    showSuccessFeedback('✅ 太棒了！任务创建成功！')
    enableNextButton()
  }
})
```

#### 3.3 操作提示
```javascript
// 如果用户 10 秒内没有操作
setTimeout(() => {
  if (!actionCompleted) {
    showHint('💡 提示：在输入框中输入任务名称，然后按回车键')
  }
}, 10000)
```
