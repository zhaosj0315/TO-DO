# 任务日志系统 - 快速测试指南

## 🚀 启动应用

```bash
# 开发模式
npm run dev

# 或者构建后预览
npm run build
npm run preview
```

## ✅ 测试步骤

### 1. 基础功能测试

#### 1.1 查看任务详情
1. 登录应用
2. 点击任意任务卡片
3. ✅ 应该看到任务详情弹窗
4. ✅ 弹窗包含：任务概览、执行统计、日志列表、操作按钮

#### 1.2 添加第一条日志
1. 在任务详情页面点击"📝 添加日志"
2. 选择日志类型："🚀 开始"
3. 输入内容："开始执行这个任务"
4. 选择耗时：30分钟
5. 点击"💾 保存日志"
6. ✅ 日志应该出现在列表中
7. ✅ 统计数据应该更新（1次推进，30分钟）

#### 1.3 添加进展日志
1. 再次点击"📝 添加日志"
2. 选择类型："📈 进展"
3. 输入内容："完成了第一部分工作"
4. 设置进度：50%
5. 添加标签："数据收集"
6. 选择心情："😊 顺利"
7. 保存
8. ✅ 日志列表应该有2条记录
9. ✅ 统计显示：2次推进，进度50%

#### 1.4 添加阻碍日志
1. 添加新日志
2. 选择类型："🚫 阻碍"
3. 输入内容："遇到技术问题，需要查资料"
4. 保存
5. ✅ 统计显示：1次阻碍（未解决）

#### 1.5 添加解决方案
1. 添加新日志
2. 选择类型："💡 方案"
3. 输入内容："找到了解决办法"
4. 在"关联之前的阻碍"下拉框中选择刚才的阻碍
5. 保存
6. ✅ 阻碍应该标记为已解决

#### 1.6 完成任务
1. 点击"✅ 标记完成"
2. 会自动打开添加日志弹窗，类型默认为"完成"
3. 输入完成总结："任务顺利完成"
4. 选择质量评分：⭐⭐⭐⭐
5. 添加经验教训：
   - "提前规划很重要"
   - "遇到问题及时求助"
6. 保存
7. ✅ 任务应该标记为已完成
8. ✅ 日志列表显示完成总结

### 2. 任务卡片显示测试

1. 返回任务列表
2. ✅ 任务卡片应该显示：
   - 💬 4条（日志数量）
   - 📊 100%（最终进度）

### 3. 数据持久化测试

1. 刷新页面
2. 再次打开任务详情
3. ✅ 所有日志应该还在
4. ✅ 统计数据正确

### 4. 多任务测试

1. 创建3个新任务
2. 为每个任务添加不同数量的日志
3. ✅ 每个任务的日志应该独立存储
4. ✅ 统计数据互不影响

## 🐛 常见问题排查

### 问题1：点击任务卡片没反应
- 检查浏览器控制台是否有错误
- 确认 `openTaskDetail` 方法是否正确调用
- 检查 `showTaskDetail` 和 `detailTask` 的值

### 问题2：日志保存后不显示
- 检查 `addTaskLog` 方法是否正确执行
- 查看浏览器 Application → Storage → Preferences
- 确认任务数据中有 `logs` 数组

### 问题3：统计数据不更新
- 检查 `calculateTaskStats` 方法是否被调用
- 确认 `stats` 对象结构正确

### 问题4：样式显示异常
- 清除浏览器缓存
- 重新构建：`npm run build`
- 检查 CSS 是否正确加载
