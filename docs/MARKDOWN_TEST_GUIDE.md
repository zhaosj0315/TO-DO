# 🧪 Markdown 编辑器快速测试指南

## 📋 测试清单

### ✅ 测试 1：全屏编辑器预览

1. 打开应用：http://localhost:5174/
2. 登录账号
3. 点击任务输入框右侧的 `⛶ 展开` 按钮
4. 输入以下测试内容：

```markdown
# 测试标题

## 子标题

这是一段**粗体文字**和*斜体文字*。

### 列表测试
- 列表项 1
- 列表项 2
- 列表项 3

### 代码测试
行内代码：`console.log('test')`

代码块：
\`\`\`javascript
function hello() {
  return 'Hello World'
}
\`\`\`

### 引用测试
> 这是一段引用文字
> 可以多行显示

### 链接测试
[GitHub](https://github.com)
```

5. 点击状态栏的 `👁️ 预览` 按钮
6. 检查渲染效果：
   - ✅ 标题层级正确
   - ✅ 粗体和斜体显示正确
   - ✅ 列表格式正确
   - ✅ 代码块有灰色背景
   - ✅ 引用有左侧边框
   - ✅ 链接是紫色
7. 点击 `✏️ 编辑` 按钮返回编辑模式
8. 确认内容没有丢失

### ✅ 测试 2：任务详情页预览

1. 创建一个新任务（使用上面的 Markdown 内容）
2. 保存任务
3. 点击任务卡片打开详情页
4. 在"📝 任务描述"区域，点击右上角的 `👁️ 预览` 按钮
5. 检查渲染效果
6. 点击预览区域或 `✏️ 编辑` 按钮返回编辑
7. 确认可以正常编辑

| 技术 | 版本 | 用途 |
|------|------|------|

| Vite | 6.x | 构建工具 |
| Node | 18.x | 运行环境 |

## 💻 代码示例

### 后端 API
\`\`\`javascript
app.get('/api/tasks', async (req, res) => {

### 前端组件
\`\`\`vue
<template>
  <div class="task-list">
    <TaskCard v-for="task in tasks" :key="task.id" />
