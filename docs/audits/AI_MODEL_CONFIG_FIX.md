# AI模型配置逻辑修复说明

## 修复的问题

### 原问题
- 每次添加模型都需要重新输入厂商地址和API Key
- 编辑模型时会丢失原始配置信息
- 无法复用已保存的厂商配置

### 新逻辑（三层架构）

```
第一层：厂商配置（持久化）
├── 类型（local/openai/custom）
├── 地址（URL）
└── API Key

第二层：模型配置（基于厂商配置）
├── 关联厂商配置ID
├── 选择的模型名称
└── 自动生成显示名称

第三层：默认模型选择
└── 选择某个模型作为默认使用
```

## 使用流程

### 1. 首次配置
1. 选择模型类型（本地Ollama/OpenAI/自定义）
2. 输入厂商地址和API Key
3. 点击"🔄 获取模型"按钮
4. **系统自动保存厂商配置**（持久化到localStorage）
5. 从下拉列表选择模型
6. 点击"➕ 添加模型"

### 2. 添加更多模型（复用配置）
1. 在"📦 使用已保存的厂商配置"下拉框中选择已有配置
2. **系统自动填充地址和API Key**
3. **系统自动获取模型列表**
4. 选择不同的模型
5. 点击"➕ 添加模型"
6. **可以继续选择其他模型，无需重新输入配置**

### 3. 编辑模型
1. 点击"编辑"按钮
2. **系统自动从厂商配置恢复地址和API Key**
3. **系统自动重新获取模型列表**
4. 修改后保存

### 4. 清空表单
- 点击"🔄 清空"按钮可以清空当前表单，重新配置

## 核心改进

1. ✅ **厂商配置持久化** - 地址和API Key只需输入一次
2. ✅ **配置复用** - 可以在同一厂商配置下添加多个模型
3. ✅ **编辑保留配置** - 编辑时自动恢复原始配置
4. ✅ **智能表单重置** - 添加模型后保留配置，可继续添加
5. ✅ **快速选择** - 下拉框快速选择已保存的配置

## 测试场景

### 场景1：添加多个Ollama模型
1. 输入Ollama地址 `http://192.168.31.159:11434/api/generate`
2. 获取模型列表
3. 选择 `gemma2:2b` → 添加
4. **无需重新输入地址**，直接选择 `llama3:8b` → 添加
5. **无需重新输入地址**，直接选择 `qwen2.5:7b` → 添加

### 场景2：添加OpenAI模型
1. 输入OpenAI地址和API Key
2. 获取模型列表
3. 选择 `gpt-4o-mini` → 添加
4. **无需重新输入API Key**，直接选择 `gpt-4o` → 添加

### 场景3：编辑模型
1. 点击某个模型的"编辑"按钮
2. **地址和API Key自动填充**
3. **模型列表自动加载**
4. 修改模型名称或选择其他模型
5. 保存

### 场景4：切换配置
1. 已有Ollama配置和OpenAI配置
2. 在下拉框选择"Ollama配置"
3. **自动填充地址，自动获取模型列表**
4. 添加模型
5. 在下拉框选择"OpenAI配置"
6. **自动填充地址和API Key，自动获取模型列表**
7. 添加模型

## 注意事项

- 厂商配置会在首次"获取模型"成功后自动保存
- 相同类型和地址的配置会自动合并（更新API Key）
- 删除模型不会删除厂商配置（可能有其他模型在使用）
- 清空表单会清除当前输入，但不会删除已保存的配置
