## 🎯 返回手势行为

### 1. 弹窗打开时
**行为**: 关闭当前弹窗（按层级优先级）

**层级顺序**:
- **第三层**（最上层）：密码、手机号、周期选择、自定义日期、番茄钟统计、AI配置、版本更新等30+个弹窗
- **第二层**（中层）：教程模式、番茄钟计时器、统一报告中心、任务详情、AI问答、个人主页等
- **子弹窗**：任务详情内的添加日志、个人主页内的统计中心等

**示例**:
```
用户操作: 左右边缘向内滑动
结果: 关闭最上层弹窗
```

---

### 2. 表单有内容时
**行为**: 逐步清空表单状态

**清空顺序**:
1. 清空任务描述
2. 清空任务标题
3. 恢复表单默认值（类型、分类、优先级等）
4. 恢复筛选状态（回到"全部"）

**示例**:
```
用户操作: 左右边缘向内滑动
状态: 任务描述有内容
结果: 清空任务描述
```

---

### 3. 首页无弹窗无表单内容时
**行为**: 退出应用到后台

**关键点**:
- ✅ **不是关闭应用**，而是最小化到后台
- ✅ **保持状态**：重新进入时停留在原页面
- ✅ **标准Android行为**：与其他应用一致

**示例**:
```
用户操作: 左右边缘向内滑动
状态: 首页，无弹窗，无表单内容
结果: 应用退到后台
重新进入: 仍然在首页
```

## 🔧 技术实现

### 核心代码
```javascript
App.addListener('backButton', ({ canGoBack }) => {
  // 1. 检查弹窗（按层级关闭）
  if (showGrowthDetail.value) {
    showGrowthDetail.value = false
    return
  }
  // ... 其他弹窗
  
  // 2. 检查表单状态（逐步清空）
  else if (newTaskDescription.value.trim() !== '') {
    newTaskDescription.value = ''
  } else if (newTaskText.value.trim() !== '') {
    newTaskText.value = ''
  }
  // ... 其他表单状态
  
  // 3. 首页状态：退出应用到后台
  else {
    App.exitApp()
  }
})
```

### 关键API
- `App.addListener('backButton', ...)` - 监听返回手势
- `App.exitApp()` - 退出应用到后台（不是关闭）
- `canGoBack` - 路由是否可返回（本应用单页面，始终为false）

## 🐛 历史问题

### 问题1: 首页返回不退出应用
**症状**: 在首页滑动返回，应用无响应

**原因**: 
```javascript
// 错误的逻辑
else if (canGoBack) {
  window.history.back()  // 单页面应用，canGoBack始终为false
} else {
  App.exitApp()  // 永远走不到这里
}
```

**修复**:
```javascript
// 正确的逻辑
else {
  App.exitApp()  // 直接退出到后台
}
```

### 问题2: 返回手势在手势导航设备上不工作
**症状**: 一加13等设备滑动返回无效

**原因**: 未正确监听 `backButton` 事件

**修复**: 已在 v0.8.1 修复
