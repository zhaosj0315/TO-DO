```javascript
task = {

  id: 456,

```

**关系特征**:
- **归属关系**: 任务属于笔记本

- **层级关系**: 笔记本可嵌套

**当前问题**:
- ✅ 数据已存储

- ❌ 无法看到笔记本内任务的聚类

**影响**:
- 无法按笔记本分组显示
- 无法看到笔记本之间的关系
- 图谱缺少重要的组织维度

## 🎯 优化方案

### 方案A：笔记本分组显示（推荐）

**目标**: 在图谱中按笔记本分组显示任务

**实现**:
1. 使用 ECharts 的 `categories` 功能
2. 每个笔记本一个分类
3. 同笔记本任务聚集在一起
4. 不同笔记本用虚线边框区分

**优点**:
- 视觉清晰
- 不增加节点数量
- 性能友好

**缺点**:
- 不显示笔记本层级

---

### 方案B：笔记本作为节点（完整）

**目标**: 笔记本作为独立节点显示

**实现**:
1. 笔记本节点（大圆形，60px）
2. 任务节点（小圆形，40px）
3. 笔记本-任务用灰色虚线连接
4. 笔记本-笔记本用蓝色实线连接（父子）

**优点**:
- 完整展示所有关系
- 可视化笔记本层级

**缺点**:
- 节点数量增加
- 图谱可能复杂

---

### 方案C：混合模式（最佳）

**目标**: 可切换的笔记本视图

**实现**:
1. 默认：按笔记本分组显示任务
2. 开关：显示/隐藏笔记本节点
3. 筛选：只显示某个笔记本的任务

**优点**:
- 兼顾简洁和完整
- 用户可按需切换
- 性能可控

## ⚡ 性能优化方案

### 问题：打开图谱慢、动画卡顿

**原因分析**:
1. 初始化动画时间过长
2. 力导向布局计算复杂
3. 节点数量多时性能下降

**优化措施**:

#### 1. 减少初始化动画时间
```javascript
force: {
  initLayout: 'circular',  // 使用圆形初始布局
  repulsion: 100,
  edgeLength: 150,
  layoutAnimation: false  // 🔧 关闭布局动画
}
```

#### 2. 延迟渲染
```javascript
onMounted(() => {
  nextTick(() => {
    setTimeout(() => {
      initChart()  // 延迟100ms初始化
    }, 100)
  })
})
```

#### 3. 虚拟化大数据
```javascript
// 超过100个任务时自动限制
if (tasks.length > 100) {
  tasks = tasks.slice(0, 100)
  showLimitWarning.value = true
}
```

#### 4. 优化力导向参数
```javascript
force: {
  repulsion: 50,        // 降低斥力（100 → 50）
  edgeLength: 100,      // 缩短边长（150 → 100）
  friction: 0.6,        // 增加摩擦力（0.3 → 0.6）
  layoutAnimation: false // 关闭动画
}
```
