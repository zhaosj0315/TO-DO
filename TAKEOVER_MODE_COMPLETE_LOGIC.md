# 接管模式完整逻辑说明

## 核心理念

**接管模式 = 数据库作为中枢，多设备实时同步**

```
本地模式（默认）：
设备A ← 本地存储 → 设备A
设备B ← 本地存储 → 设备B
（数据隔离，不互通）

接管模式（开启后）：
设备A ↔ 数据库（中枢） ↔ 设备B
（数据实时同步，完全一致）
```

---

## 数据流向

### 1. 本地优先原则（不变）

```
用户操作
  ↓
本地存储（Preferences + localStorage）← 主数据源（立即保存）
  ↓
应用界面更新（立即响应）
```

**保证**：
- ✅ 离线可用
- ✅ 即时响应
- ✅ 数据安全

### 2. 双向同步机制（接管模式）

#### 上行同步：本地 → 数据库

```javascript
// 每次数据变更时自动触发
saveTasks() {
  // 1. 保存到本地（阻塞，必须成功）
  await Preferences.set(...)
  
  // 2. 自动备份（阻塞）
  await performBackup()
  
  // 3. 同步到数据库（异步，不阻塞）
  this.syncToDatabase()  // 不await
}

syncToDatabase() {
  if (接管模式开启) {
    // 上传到数据库
    await mysqlSyncService.syncToMySQL(...)
  }
}
```

**触发时机**：
- ✅ 创建任务
- ✅ 编辑任务
- ✅ 完成任务
- ✅ 删除任务
- ✅ 添加日志
- ✅ 创建文件夹
- ✅ 移动任务
- ✅ 所有数据变更操作

#### 下行同步：数据库 → 本地

```javascript
// 应用启动时自动触发

  // 1. 从数据库拉取最新数据（如果开启接管）
  await this.pullFromDatabase()
  
  // 2. 加载本地数据
  const tasks = await Preferences.get(...)
  this.tasks = JSON.parse(tasks)
}

pullFromDatabase() {
  if (接管模式开启) {
    // 从数据库下载最新数据
    await mysqlSyncService.restoreFromMySQL(...)
    // 自动合并到本地（智能去重）
  }
}
```

**触发时机**：
- ✅ 应用启动
- ✅ 用户登录
- ✅ 手动刷新

## 数据合并策略

### 智能去重算法

```javascript
mergeByIdKeepLatest(localData, remoteData) {
  const map = new Map()
  
  // 1. 添加本地数据
  localData.forEach(item => {
    map.set(item.id, item)
  })
  
  // 2. 添加远程数据
  remoteData.forEach(item => {
    const existing = map.get(item.id)
    if (!existing) {
      // 不存在，直接添加（远程新数据）
      map.set(item.id, item)
    } else {
      // 存在，比较时间戳
      const localTime = new Date(existing.updated_at || existing.created_at).getTime()
      const remoteTime = new Date(item.updated_at || item.created_at).getTime()
      
      if (remoteTime > localTime) {
        // 远程更新，覆盖本地
        map.set(item.id, item)
      }
      // 否则保留本地（本地更新）
    }
  })
  
  return Array.from(map.values())
}
```

### 合并规则

| 情况 | 本地 | 远程 | 结果 |
|------|------|------|------|
| 新数据 | 无 | 有 | 添加远程数据 |
| 新数据 | 有 | 无 | 保留本地数据 |
| 相同ID | 旧版本 | 新版本 | 使用远程数据 |
| 相同ID | 新版本 | 旧版本 | 保留本地数据 |
| 相同ID | 相同版本 | 相同版本 | 保留任意一个 |

## 接管模式使用流程

### 首次配置（设备A）

1. **配置数据库**
   ```
   打开"数据库配置"
   → 选择MySQL
   → 输入数据库信息
   → 测试连接
   → 保存配置
   ```

2. **开启接管模式**
   ```
   点击"接管模式"按钮
   → 按钮变为"✅ 已接管"
   → 自动上传所有数据到数据库
   ```

3. **正常使用**
   ```
   创建任务 → 自动同步到数据库
   编辑任务 → 自动同步到数据库
   所有操作 → 自动同步到数据库
   ```

### 新设备接入（设备B）

1. **登录账号**
   ```
   使用相同的账号登录
   ```

2. **配置数据库**
   ```
   输入相同的数据库信息
   → 测试连接
   → 保存配置
   ```

3. **开启接管模式**
   ```
   点击"接管模式"按钮
   → 按钮变为"✅ 已接管"
   → 自动拉取数据库中的所有数据
   → 自动合并到本地
   ```

4. **自动同步**
   ```
   每次打开应用 → 自动拉取最新数据
   每次操作 → 自动上传到数据库
   ```
