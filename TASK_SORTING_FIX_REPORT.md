## 🔴 P0 级别修复（已完成）

### P0-1: 已完成任务排序的 copy-paste 错误 ✅
**问题**: `bTime` 计算中错误地使用了两次 `b.completed_at`  
**影响**: 当 `b.completed_at` 不存在时，`bTime = NaN`，导致排序失效  
**修复**: 将 `b.completed_at` 改为 `b.created_at`（Line 967）

### P0-2: 重复任务截止时间计算验证 ✅
**验证结果**: `calculateDeadline` 方法已正确处理所有重复任务类型
- `daily`: 当天 23:59:59
- `weekday`: 当天 23:59:59  
- `weekly`: 下一个指定星期几的 23:59:59

## 🟠 P1 级别修复（已完成）

### P1-1: 2小时紧急预警跨越优先级边界 ✅
**问题**: 低优先级任务在2小时内到期时（权重640）会排在高优先级正常任务（权重700）之前

**修复前**:
```
低优先级 + 2小时内 = 640
高优先级 + 正常 = 700
结果: 低优先级排在前面 ❌
```

**修复后**:
```
高优先级正常: 700-799
高优先级2小时内: 650-699（在高优先级范围内提升）
中优先级正常: 800-899
中优先级2小时内: 750-799（不会超越高优先级）
低优先级正常: 900-999
低优先级2小时内: 850-899（不会超越中优先级）
```

**核心改进**: 紧急预警在同优先级内提升50分，不跨越优先级边界

---

### P1-2: 逾期衰减算法的权重跳跃 ✅
**问题1**: 逾期3天前后权重突变（100 → 231）  
**问题2**: 低优先级逾期任务惩罚过重  
**问题3**: 逾期30天后权重不再增长

**修复前**:
```
逾期2.9天高优先级: 100
逾期3.1天高优先级: 231（突变131分）
逾期4天低优先级: 340
逾期4天高优先级: 240（低优先级反而更靠前）
```

**修复后**:
```
高优先级逾期: 100-250（每天+5分，连续衰减）
  - 逾期1天: 105
  - 逾期3天: 115
  - 逾期30天: 250

中优先级逾期: 150-300（每天+5分）
  - 逾期1天: 155
  - 逾期30天: 300

低优先级逾期: 200-350（每天+5分）
  - 逾期1天: 205
  - 逾期30天: 350
```

**核心改进**: 
1. 连续线性衰减，无跳跃
2. 高优先级始终排在低优先级前面
3. 30天封顶后权重稳定
