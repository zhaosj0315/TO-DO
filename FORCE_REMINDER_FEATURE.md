# 强制提醒功能说明

## 功能概述
强制提醒是一种更强力的提醒方式，适用于重要任务。与普通提醒相比，强制提醒具有以下特点：

## 核心特性

### 1. 不可滑动关闭
- ✅ 通知设置为`ongoing=true`
- ✅ 无法通过滑动手势关闭
- ✅ 必须点击通知进入应用处理

### 2. 点击不自动消失
- ✅ 通知设置为`autoCancel=false`
- ✅ 点击通知后不会自动消失
- ✅ 需要在应用内手动关闭

### 3. 持续响铃（Android限制）
- ⚠️ Android系统限制：通知铃声只响一次
- ⚠️ 无法实现真正的"持续响铃"
- ✅ 使用最高优先级确保铃声响起

## 使用方法

### 创建强制提醒任务
1. 输入任务名称
2. ✅ 勾选"🔔 启用提醒"
3. 选择提醒时间
4. ✅ 勾选"🔊 强制提醒"
5. 点击"✓"提交

### 强制提醒标识
- 任务列表中显示🔊图标
- 提醒时通知无法滑动关闭
- 必须进入应用处理

## 技术实现

### 通知配置
```javascript
{
  ongoing: true,        // 不可滑动关闭
  autoCancel: false,    // 点击不消失
  sound: 'default',     // 系统铃声
  channelId: 'task-reminders-v3',
  extra: {
    forceReminder: true,
    priority: 'max',
    fullScreen: true
  }
}
```

### 数据结构
```javascript
task: {
  enableReminder: Boolean,  // 是否启用提醒
  reminderTime: String,     // 提醒时间
  forceReminder: Boolean    // 是否强制提醒
}
```

## Android系统限制

### 无法实现的功能
1. ❌ **持续响铃**：Android不允许通知持续播放声音
2. ❌ **全屏弹窗**：需要额外的权限和Activity
3. ❌ **强制唤醒屏幕**：需要WAKE_LOCK权限

### 可实现的功能
1. ✅ **不可滑动关闭**：ongoing通知
2. ✅ **点击不消失**：autoCancel=false
3. ✅ **最高优先级**：importance=5
4. ✅ **震动提醒**：持续震动模式

## 替代方案

如果需要真正的"持续响铃"，需要：
1. 创建独立的Activity
2. 使用MediaPlayer循环播放铃声
3. 全屏显示关闭按钮
4. 需要额外的权限配置

这需要更复杂的原生Android开发，超出了Capacitor插件的能力范围。

## 建议

对于重要任务：
- ✅ 使用强制提醒（不可滑动关闭）
- ✅ 设置多个提醒时间
- ✅ 提高任务优先级
- ✅ 使用震动模式

## 注意事项

1. 强制提醒会占用通知栏空间
2. 用户必须进入应用才能关闭
3. 不要滥用强制提醒功能
4. 建议只用于真正重要的任务
