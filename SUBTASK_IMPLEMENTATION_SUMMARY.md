## 🎯 核心函数说明

### 1. `detectSubtasksFromDescription(description)`
**功能**：从描述中检测列表项

**输入**：任务描述字符串

**输出**：子任务名称数组

**逻辑**：
1. 定义5种列表格式的正则表达式
2. 遍历所有格式进行匹配
3. 提取匹配到的任务名称
4. 验证长度（1-100字）
5. 去重并返回

**示例**：
```javascript
const description = `
完成项目报告
1. 收集数据
2. 分析结果
3. 撰写报告
`

detectSubtasksFromDescription(description)
// 返回: ['收集数据', '分析结果', '撰写报告']
```

### 2. `acceptDetectedSubtasks()`
**功能**：采纳自动识别的子任务

**逻辑**：
1. 将检测到的子任务转换为手动子任务格式
2. 继承主任务的分类和优先级
3. 关闭建议气泡
4. 显示成功提示

### 3. `toggleManualSubtasks()`
**功能**：切换手动子任务管理界面

**逻辑**：
1. 切换显示状态
2. 如果打开且有检测到的子任务，自动填充
3. 如果关闭，保留已添加的子任务

### 4. `addManualSubtask()`
**功能**：添加单个子任务

**逻辑**：
1. 验证输入（非空、长度限制）
2. 创建子任务对象（包含text、priority、category）
3. 添加到子任务列表
4. 清空输入框
5. 显示成功提示

### 5. `removeManualSubtask(index)`
**功能**：删除指定子任务

**逻辑**：
1. 从子任务列表中移除指定索引的项
2. 显示删除提示

### 6. `clearAllSubtasks()`
**功能**：清空所有子任务

**逻辑**：
1. 清空子任务列表
2. 显示清空提示

### 7. `addTask()` (修改)
**功能**：创建主任务和子任务

**新增逻辑**：
1. 创建主任务
2. 检查是否有子任务
3. 如果有，循环创建子任务：
   - 添加序号：`1. 子任务名`
   - 生成描述：`子任务 1/4（父任务：XXX）`
   - 设置依赖：`waitFor: previousSubtaskId`
   - 继承属性：分类、优先级、日期时间
4. 清空子任务列表
5. 显示成功提示

## 🔄 用户交互流程

### **流程1：自动识别（被动触发）**

```
用户操作                    系统响应
─────────────────────────────────────────
1. 点击"⛶ 展开编辑"      → 打开全屏编辑
2. 输入任务标题           → 显示标题
3. 输入任务描述（含列表）  → 显示描述
4. 点击"完成"             → 检测列表项
                           → 弹出建议气泡
5. 点击"✓ 创建子任务"     → 转换为手动子任务
                           → 关闭气泡
6. 点击主任务"✓"提交      → 创建主任务和子任务
                           → 设置依赖关系
                           → 显示成功提示
```

### **流程2：手动添加（主动触发）**

```
用户操作                    系统响应
─────────────────────────────────────────
1. 点击"⛶ 展开编辑"      → 打开全屏编辑
2. 点击"🌳 子任务"按钮    → 滑出管理面板
3. 输入子任务名称         → 显示输入内容
4. 按回车或点击"+"        → 添加到列表
                           → 显示序号和删除按钮
5. 重复步骤3-4            → 继续添加子任务
6. 点击"完成"             → 关闭全屏编辑
7. 点击主任务"✓"提交      → 创建主任务和子任务
                           → 设置依赖关系
                           → 显示成功提示
```
