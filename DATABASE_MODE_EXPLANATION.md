# 数据库模式说明

## 三种模式

### 1. 本地模式（默认）✅
```
图标：📱
名称：本地存储
说明：仅使用应用内存储
特点：
  - ✅ 默认模式
  - ✅ 完全离线
  - ✅ 无需配置
  - ✅ 数据存储在应用内部
  - ✅ 自动保存
```

### 2. SQLite模式（可选）
```
图标：💾
名称：SQLite
说明：本地数据库备份（仅Android/iOS）
特点：
  - 📱 仅支持原生应用
  - 💾 本地数据库文件
  - 🔄 可开启接管模式
  - ❌ 网页端不支持
```

### 3. MySQL模式（可选）
```
图标：🌐
名称：MySQL
说明：远程服务器备份
特点：
  - 🌐 支持所有平台
  - ☁️ 云端数据库
  - 🔄 可开启接管模式
  - 📱 多设备同步
```

---

## 默认行为

### 首次使用
```
1. 用户打开应用
2. 默认使用"本地模式"
3. 数据存储在应用内部
4. 无需任何配置
5. 完全离线可用
```

### 数据库类型选择
```
用户可以在"数据库配置"中选择：
- 📱 本地存储（默认）
- 💾 SQLite（可选）
- 🌐 MySQL（可选）
```

---

## 代码逻辑

### 默认值
```javascript
const dbType = ref('none')  // 默认本地模式
```

### 同步逻辑
```javascript
async syncToDatabase() {
  const { value: dbType } = await Preferences.get({ key: 'db_type' })
  
  if (dbType === 'mysql') {
    // MySQL同步
  } else if (dbType === 'sqlite') {
    // SQLite同步
  }
  // 如果 dbType 为空或 'none'，不执行任何操作
}
```

### 拉取逻辑
```javascript
async pullFromDatabase() {
  const { value: dbType } = await Preferences.get({ key: 'db_type' })
  
  if (dbType === 'mysql') {
    // 从MySQL拉取
  }
  // 如果 dbType 为空或 'none'，不执行任何操作
}
```

## 用户体验

### 本地模式（默认）
```
✅ 打开应用 → 直接使用
✅ 创建任务 → 自动保存到本地
✅ 完全离线 → 无需网络
✅ 零配置 → 开箱即用
```

### SQLite模式（可选）
```
1. 打开"数据库配置"
2. 选择"SQLite"
3. 点击"测试连接"
4. 点击"保存配置"
5. 开启"接管模式"（可选）
```

### MySQL模式（可选）
```
1. 打开"数据库配置"
2. 选择"MySQL"
3. 输入数据库信息
4. 点击"测试连接"
5. 点击"保存配置"
6. 开启"接管模式"（可选）
```

---

## 总结

✅ **默认：本地模式**
- 无需配置
- 完全离线
- 自动保存

✅ **可选：SQLite/MySQL**
- 用户自行选择
- 需要配置
- 可开启接管模式

✅ **逻辑正确**
- 默认值是 'none'
- 只有选择数据库后才会同步
- 本地模式不执行任何数据库操作
