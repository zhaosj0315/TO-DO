## 🎯 核心更新

### 1. 任务置顶功能
用户可以手动控制任务排序优先级，无论任务状态如何，置顶任务始终显示在列表最前面。

**功能特性**:
- 📌 任务卡片右上角新增置顶按钮
- 🎨 视觉反馈：未置顶（灰色半透明）→ 已置顶（彩色旋转45度）
- 🔝 置顶任务永远在最前面，内部按优先级细分

**使用场景**:
```
场景：临时紧急任务
任务：明天面试准备材料（中优先级）
操作：点击 📌 置顶
结果：即使有高优先级任务，这个任务也会排在最前面
```

### 2. 智能权重排序系统
重构排序逻辑为基于权重的智能算法，解决三大痛点：
1. ❌ 逾期任务淹没首屏
2. ❌ 紧急但不逾期的任务被忽略
3. ❌ 置顶任务通货膨胀

**权重分层体系**:
```
0-2      置顶区（按优先级细分：高0/中1/低2）
100-200  逾期高危区（3天内逾期）
200-500  逾期衰减区（3天以上，低优先级降权）
600-650  紧急预警区（2小时内到期）
700-900  正常待办区（按优先级：高700/中800/低900）
10000+   已完成区（按完成时间倒序）
```

**智能机制**:
- ⏰ **2小时紧急预警**: 距离截止时间不足2小时的任务自动提升到紧急预警区
- 📉 **逾期惩罚衰减**: 逾期3天以上的低优先级任务权重降低，避免淹没首屏
- 🎯 **置顶区细分**: 置顶任务内部严格按优先级排序，避免通货膨胀

**排序示例**:
```
任务A：低优先级，逾期7天 → 权重 500（衰减区）
任务B：高优先级，1小时后到期 → 权重 600（紧急预警）
任务C：中优先级，明天到期，已置顶 → 权重 1（置顶区）

排序结果：C(1) → B(600) → A(500) ✅
```

## 🔧 技术实现

### Store 层改动
**文件**: `src/stores/offlineTaskStore.js`

1. **数据模型扩展**:
```javascript
// addTask 方法新增字段
is_pinned: taskData.is_pinned || false
```

2. **新增方法**:
```javascript
async togglePin(taskId) {
  const task = this.tasks.find(t => t.id === taskId)
  if (task) {
    task.is_pinned = !task.is_pinned
    await this.saveTasks()
  }
}
```

3. **排序逻辑重构**:
- 从4级排序改为权重计算模型
- 动态计算截止时间，基于时间差分配权重
- 逾期任务根据逾期天数和优先级计算惩罚权重

### UI 层改动
**文件**: `src/views/TodoView.vue`

1. **任务卡片结构调整**:
```vue
<div class="task-actions">
  <button class="btn-pin-inline" :class="{ 'pinned': task.is_pinned }">
    📌
  </button>
  <button class="btn-delete-inline">🗑️</button>
</div>
```

2. **新增样式**:
- `.task-actions`: 按钮组容器
- `.btn-pin-inline`: 置顶按钮（灰度/彩色/旋转状态）

## 📝 使用建议

### 置顶功能最佳实践
1. **控制数量**: 建议不超过3个置顶任务，避免失去焦点
2. **临时使用**: 用于临时紧急任务，完成后及时取消置顶
3. **配合优先级**: 置顶区内部仍按优先级排序，合理设置优先级

### 智能排序理解
- 系统会自动识别紧急任务（2小时内到期）
- 逾期任务会根据逾期时长和优先级动态调整位置
- 不需要手动干预，系统自动优化排序
