## 🔄 数据流

```
用户操作 → 添加日志
    ↓
Store.addTaskLog()
    ↓
1. 创建日志对象（自动添加时间戳）
2. 添加到 task.logs 数组
3. 调用 calculateTaskStats() 重新计算统计
4. 如果是完成类型，自动标记任务为完成
5. 保存到本地存储
    ↓
UI 自动更新
    ↓
任务卡片显示新的日志数量和进度
```

## 💡 设计决策

### 1. 为什么选择 Bottom Sheet？
- 符合移动端操作习惯
- 不完全遮挡背景，保持上下文
- 滑出动画更自然

### 2. 为什么日志类型必选？
- 强制用户思考当前操作的性质
- 便于后续统计和分析
- 提供更结构化的数据

### 3. 为什么时间自动记录？
- 减少用户输入负担
- 保证时间准确性
- 简化操作流程

### 4. 为什么统计自动计算？
- 保证数据一致性
- 避免手动维护统计
- 实时反馈执行情况

### 5. 为什么保留快速完成？
- 不强制用户添加日志
- 保持原有简单流程
- 给用户选择权

## 🎓 技术亮点

1. **响应式设计**
   - 使用 Vue 3 Composition API
   - computed 自动计算排序后的日志
   - watch 监听数据变化

2. **组件化**
   - 任务详情和添加日志独立组件
   - 通过 props 和 emit 通信
   - 可复用性强

3. **数据迁移**
   - 在 loadTasks 时自动迁移
   - 不影响现有数据
   - 向后兼容

4. **类型安全**
   - 日志类型枚举
   - 数据结构清晰
   - 便于维护

5. **用户体验**
   - 表单验证
   - 字符计数
   - 快速选择
   - 动画过渡

## 📝 使用示例

### 场景1：简单任务
```
1. 创建任务："买菜"
2. 点击任务卡片
3. 添加日志："开始" - "去超市"
4. 添加日志："完成" - "买完了"
```

```
1. 创建任务："完成季度报告"
2. 添加日志："开始" - "收集数据" - 耗时30分钟
3. 添加日志："进展" - "完成数据整理" - 进度30% - 标签"数据"
4. 添加日志："阻碍" - "统计公式出错"
5. 添加日志："方案" - "找到解决办法" - 关联阻碍
6. 添加日志："里程碑" - "数据分析完成" - 进度80%
7. 添加日志："完成" - "报告完成" - 评分4星 - 经验教训
8. 查看统计：6次推进，X小时，1次阻碍已解决
```
