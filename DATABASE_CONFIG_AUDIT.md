## ❌ 发现的漏洞

### 漏洞1: SQLite同步缺少deletedTasks ⚠️
**位置**: `syncNow()` 函数
```javascript
// 当前代码
const result = await sqliteService.sync(
  userData.username, 
  userData.tasks,        // ✅ 有
  userData.collections   // ✅ 有
  // ❌ 缺少 deletedTasks
)
```

**影响**: SQLite同步时不会备份回收站数据

**修复方案**:
```javascript
const result = await sqliteService.sync(
  userData.username, 
  userData.tasks,
  userData.deletedTasks,  // ✅ 添加
  userData.collections
)
```

### 漏洞2: 重复的配置加载 ⚠️
**位置**: `onMounted()` 函数
```javascript
// 第1次加载
const savedMysql = await mysqlConfigService.getConfig()
if (savedMysql) {
  mysqlConfig.value = savedMysql
  // ... 测试连接
}

// 第2次加载（重复）
const savedMysql = await mysqlConfigService.getConfig()
if (savedMysql) {
  mysqlConfig.value = savedMysql
}
```

**影响**: 性能浪费，逻辑冗余

**修复方案**: 删除第二次加载

### 漏洞3: SQLite恢复功能缺失 ⚠️
**位置**: `restoreData()` 函数
```javascript
// 当前代码只支持MySQL恢复
const result = await mysqlSyncService.restoreFromMySQL(...)
```

**影响**: SQLite用户无法恢复数据

**修复方案**: 添加SQLite恢复逻辑
