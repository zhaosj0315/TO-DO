# 统计功能合并方案

## 现状分析

### 1. 数据统计（DataStatsModal）- 右上角按钮
**位置**：右上角独立按钮
**内容**：
- 📈 任务完成趋势图（折线图）
- 🥧 任务分类占比（饼图）
- ⚡ 优先级分布（柱状图）
- 🗓️ 365天完成热力图
- ⭐ 任务执行质量分析（4个卡片）
  - 💬 平均日志数
  - ⚠️ 平均阻碍数
  - ⭐ 平均完成评分
  - 📊 平均进度
- 💡 效率分析（4个卡片）
  - ⏱️ 平均完成时间
  - 🎯 完成率
  - 🔥 最高效时段
  - 📊 总任务数

### 2. 番茄钟统计（showPomodoroStats）- 个人主页
**位置**：个人主页 → 番茄钟统计
**内容**：
- ⏱️ 今日专注时长（分钟）
- 🍅 今日完成番茄钟数
- 📅 本周完成番茄钟数
- 📊 番茄钟历史记录列表
- 🍅 已获得番茄（完成任务）
- 🍅 待获得番茄（待完成任务）
- 🍅 逾期扣除（逾期任务）
- 🍅 净获得番茄

## 合并方案：统一数据中心

### 设计原则
1. **取其精华**：保留两者最有价值的数据
2. **去其糟粕**：删除重复或低价值的统计
3. **统一入口**：合并到个人主页，删除右上角按钮
4. **分类展示**：按功能模块分组，清晰易读

### 优势分析

#### 优点
1. **统一入口**：所有统计数据集中在一个地方
2. **减少干扰**：右上角按钮减少，界面更简洁
3. **逻辑清晰**：个人主页 = 个人数据中心
4. **易于扩展**：Tab 结构方便未来添加新统计
5. **提升价值**：个人主页功能更丰富，用户粘性更强

#### 缺点
1. **访问路径变长**：需要先打开个人主页
   - 解决方案：可以在首页添加快捷入口（如统计卡片可点击）
2. **内容较多**：单个弹窗内容丰富
   - 解决方案：Tab 分类 + 懒加载图表

### 技术实现要点

1. **Tab 切换**：使用 `ref` 管理当前 Tab
2. **图表懒加载**：只渲染当前 Tab 的图表，提升性能
3. **数据复用**：复用 `DataStatsModal` 的图表逻辑
4. **样式统一**：使用 Bottom Sheet 布局
5. **返回手势**：支持 Android 返回手势

### 命名建议

- 组件名：`DataCenterModal.vue`
- 变量名：`showDataCenter`
- 按钮文字：`📊 数据中心`
- 弹窗标题：`📊 数据中心`

## 用户体验优化

1. **首次打开**：默认显示"概览" Tab
2. **记忆功能**：记住用户上次查看的 Tab
3. **快捷跳转**：从首页统计卡片可直接跳转到对应 Tab
4. **加载提示**：图表加载时显示 LoadingSpinner
5. **空数据提示**：无数据时显示友好提示

## 总结

这个合并方案将两个独立的统计功能整合为一个统一的"数据中心"，放在个人主页中。通过 Tab 分类展示，既保留了所有有价值的数据，又提升了界面的简洁性和逻辑性。

**核心价值**：
- 📊 统一数据入口
- 🎯 清晰的信息架构
- 🚀 更好的用户体验
- 🧹 更简洁的界面
