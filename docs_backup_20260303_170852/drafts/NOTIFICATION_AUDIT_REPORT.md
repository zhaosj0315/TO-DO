# 应用通知审查报告

**审查日期**: 2026-02-23  
**当前版本**: v1.6.11  
**通知类型数量**: 6种

---

## 📱 通知类型汇总

本应用共有 **6种通知类型**，分为3类：

### 第一类：短信验证码通知（3种）
1. 登录短信验证码 (ID: 1)
2. 注册短信验证码 (ID: 3)
3. 绑定手机短信验证码 (ID: 2)

### 第二类：任务提醒通知（2种）
4. 任务即将逾期提醒 (ID: task.id)
5. 任务已逾期提醒 (ID: task.id + 100000)

### 第三类：每日摘要通知（1种）⭐ v1.6.11新增
6. 每日任务摘要 (ID: 999999)

---

## 详细说明

### 1. 系统通知（LocalNotifications）

#### 1.1 短信验证码通知 📱
**位置**: `LoginView.vue`  
**触发场景**:
- 注册时绑定手机号
- 登录时使用手机号登录
- 个人主页绑定手机号

**通知内容**:
```javascript
{
  title: '验证码',
  body: `您的验证码是：${code}`,
  id: 1 (登录) / 2 (绑定) / 3 (注册),
  schedule: { at: new Date(Date.now() + 100) }
}
```

**频率**: 用户主动触发（点击"发送验证码"按钮）

---

#### 1.2 任务截止提醒 ⏰
**位置**: `TodoView.vue` - `checkAndNotifyDeadline()`  
**触发场景**:
- 任务即将在 1 小时内逾期
- 任务已逾期但未标记

**通知内容**:

**即将逾期（1小时内）**:
```javascript
{
  title: `⏰ ${任务名称}`,
  body: `还剩 X 分钟！🍅 番茄要逃跑啦！快来抓住它！`,
  id: task.id
}
```

**已逾期**:
```javascript
{
  title: `❌ ${任务名称}`,
  body: `已逾期 X 小时！🍅 番茄已经逃跑了...`,
  id: task.id + 100000
}
```

**防刷屏机制**: 
- 使用 `notifiedTasks` Set 记录已通知的任务
- 每个任务每种状态只通知一次
- 任务完成或删除后清除记录

**检查频率**: 每分钟检查一次（`setInterval(checkAndNotifyDeadline, 60000)`）

---

#### 1.3 每日任务摘要通知 📅 ⭐ v1.6.11新增
**位置**: `TodoView.vue` - `scheduleDailySummaryNotification()`  
**触发时间**: 每天早上 **9:00**

**通知内容**:
```javascript
{
  title: '📅 今日任务摘要',
  body: `📋 待办: ${pending}个 | ⏰ 即将逾期: ${urgent}个 | ❌ 已逾期: ${overdue}个\n${message}`,
  id: 999999
}
```

**智能激励语**:
- 无任务：`🎉 今天没有待办任务，享受轻松的一天吧！`
- 有紧急任务：`⚡ 有任务即将逾期，记得及时完成哦！`
- 有逾期任务：`❗ 有任务已经逾期，赶紧处理吧！`
- 正常情况：`💪 加油完成今天的任务吧！`

**更新机制**:
- 任务增删改时自动重新调度
- 确保通知内容始终是最新的

**技术实现**:
```javascript
await LocalNotifications.schedule({
  notifications: [{
    title: '📅 今日任务摘要',
    body: summaryText,
    id: 999999,
    schedule: {
      at: tomorrowNineAM,
      allowWhileIdle: true
    }
  }]
})
```

### 2. Toast 通知（应用内）

**位置**: `TodoView.vue` - `showToast()`  
**触发场景**: 用户操作反馈

**通知类型**:
- ✅ 成功：`任务已添加` / `任务已完成` / `任务已删除`
- ❌ 错误：`请输入任务名称` / `导出失败`
- ℹ️ 信息：`数据已导出` / `数据已导入`

## ⚠️ 潜在问题

### 1. 短信验证码无防刷屏机制 ⚠️
**问题**: 用户可以无限次点击"发送验证码"  
**风险**: 
- 通知轰炸
- 用户体验差

**建议**: 添加 60 秒倒计时限制

---

### 2. 任务提醒可能过多 ⚠️
**场景**: 如果用户有 10 个任务同时在 1 小时内逾期  
**结果**: 会收到 10 条通知

**建议**: 
- 合并通知：`您有 10 个任务即将逾期`
- 或限制单次最多 3 条通知
