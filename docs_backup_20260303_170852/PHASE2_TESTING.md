### 测试 2：添加子任务

```
步骤：
1. 打开任意任务详情
2. 点击"➕ 添加子任务"按钮
3. 填写子任务信息：
   - 任务名称：测试子任务
   - 优先级：中
   - 类型：今天
4. 点击"✓ 添加子任务"
5. 应该看到成功提示
6. 关闭详情，刷新页面
7. 再次打开任务详情，应该看到子任务进度条
```

**预期结果**：
- 弹窗从底部滑出
- 表单字段完整
- 提交成功后显示提示
- 子任务进度条显示 0/1 (0%)

### 测试 3：添加依赖关系

```
步骤：
1. 创建两个任务：
   - 任务 A：设计数据库
   - 任务 B：编写代码
2. 打开任务 B 的详情
3. 点击"🔗 添加依赖关系"按钮
4. 在列表中勾选"设计数据库"
5. 点击"✓ 添加依赖 (1)"
6. 应该看到成功提示
7. 刷新页面，再次打开任务 B 详情
```

**预期结果**：
- 看到 🔒 被阻塞提示卡片
- 前置任务列表显示"设计数据库"
- 任务 B 的 canStart 为 false

### 测试 4：完成前置任务解锁后置任务

```
步骤：
1. 完成任务 A（设计数据库）
2. 打开任务 B 的详情
3. 查看依赖状态
```

**预期结果**：
- 🔒 被阻塞提示消失
- 前置任务显示 ✅ 已完成
- 任务 B 的 canStart 变为 true

### 测试 5：循环依赖检测

```
步骤：
1. 创建任务 A、B、C
2. A 依赖 B

4. 尝试让 C 依赖 A（会形成环）
```

**预期结果**：
- 在添加依赖弹窗中，任务 A 显示为禁用状态
- 显示"⚠️ 会形成循环依赖"警告
- 无法勾选任务 A

## 🐛 可能遇到的问题

### 问题 1：组件未显示
**原因**：可能是缓存问题  
**解决**：强制刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）

### 问题 2：点击按钮无反应
**原因**：事件处理器未正确绑定  
**解决**：检查浏览器控制台是否有错误信息

### 问题 3：弹窗样式异常
**原因**：CSS 冲突  
**解决**：检查是否有其他样式覆盖了 bottom-sheet 样式

### 问题 4：数据未保存
**原因**：Phase 1 数据迁移未完成  
**解决**：
```javascript
// 在控制台执行
const taskStore = useOfflineTaskStore()
console.log('第一个任务:', taskStore.tasks[0])
// 检查是否有 parentId, childrenIds, dependencies 等字段
```
