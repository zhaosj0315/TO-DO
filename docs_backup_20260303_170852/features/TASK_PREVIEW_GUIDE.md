# 👁️ 任务预览功能使用指南

**版本**: v0.7.9  
**发布日期**: 2026-03-02

## 📋 功能概述

任务预览功能提供**双按钮模式**，让用户在创建任务时可以选择两种不同的工作流程：
1. **快速输入模式**（完成按钮）：快速输入后返回首页调整属性
2. **预览保存模式**（预览按钮）：预览、编辑、AI拆分后一次性保存

---

## 🎯 功能定位

### **灵活的任务创建工作流**

```
全屏编辑 → 双按钮选择 → 两种工作流
                    ├─ 完成：返回首页 → 调整属性 → 提交
                    └─ 预览：预览弹窗 → 编辑/拆分 → 保存
```

---

## 🚀 使用方法

### **1. 快速输入模式（完成按钮）**

**适用场景**：
- 快速记录任务想法
- 需要在首页调整多个属性
- 不确定任务的最终形态

**操作流程**：
1. 点击首页"➕"按钮，打开全屏编辑
2. 输入任务标题和描述
3. 点击导航栏"**完成**"按钮
4. 返回首页，任务输入框已填充标题和描述
5. 在首页调整分类、优先级、日期等属性
6. 点击"✓"提交按钮创建任务

**优点**：
- 快速记录，不打断思路
- 可以在首页看到任务列表的同时调整属性
- 灵活性高，可以随时修改

**示例**：
```
1. 全屏编辑输入：
   标题：完成项目报告
   描述：包括数据分析和结论

2. 点击"完成"返回首页

3. 在首页调整：
   分类：工作
   优先级：高
   日期：明天

4. 点击"✓"提交
```

---

### **2. 预览保存模式（预览按钮）**

**适用场景**：
- 需要查看任务的完整信息
- 需要使用AI拆分功能
- 确定任务形态后一次性保存

**操作流程**：
1. 点击首页"➕"按钮，打开全屏编辑
2. 输入任务标题和描述
3. 点击导航栏"**预览**"按钮
4. 在预览弹窗中：
   - 查看任务的所有属性
   - 编辑任务信息
   - 使用"🤖 AI拆分"创建子任务
5. 点击"✅ 确认保存"按钮
6. 父任务和所有子任务一次性创建

**优点**：
- 一次性保存，避免多次操作
- 支持AI拆分，自动创建子任务
- 原子性操作，父任务和子任务同时创建

**示例**：
```
1. 全屏编辑输入：
   标题：完成项目报告
   描述：
   1. 收集数据
   2. 分析结果
   3. 撰写报告

2. 点击"预览"打开预览弹窗

3. 在预览弹窗中：
   - 查看任务信息
   - 点击"🤖 AI拆分"
   - 选择"快速拆分"模板
   - 生成3个子任务

4. 点击"✅ 确认保存"

5. 结果：
   - 父任务：完成项目报告
   - 子任务1：收集数据
   - 子任务2：分析结果
   - 子任务3：撰写报告
```

## 🎨 UI界面说明

### **全屏编辑导航栏**

```
┌─────────────────────────────────────────┐
│ [任务标题输入框]    [完成] [预览]       │
└─────────────────────────────────────────┘
```

- **完成按钮**：灰色文字，次要按钮样式
- **预览按钮**：紫色渐变背景，主要按钮样式

### **预览弹窗**

预览弹窗复用**TaskDetailModal**组件，包含以下功能：
- 任务概览（状态、优先级、分类、截止时间）
- 任务描述
- 执行统计（如果有日志）
- 操作按钮：
  - **✅ 确认保存**（预览模式专属）
  - **🤖 AI拆分**
  - **✏️ 编辑**
  - **❌ 关闭**

**注意**：预览模式下，以下按钮会被隐藏：
- ✓ 标记完成
- 🗑️ 删除任务

## 🔧 技术实现

### **核心函数**

#### 1. `previewTask()`
**功能**：创建预览任务对象并打开预览弹窗

**逻辑**：
1. 验证任务标题非空
2. 解析日期时间
3. 创建临时任务对象（包含所有属性）
4. 打开预览弹窗

#### 2. `saveTaskFromPreview()`
**功能**：从预览模式保存任务（原子性操作）

**逻辑**：
1. 创建父任务
2. 如果有子任务，逐个创建子任务
3. 更新父任务的子任务ID列表
4. 清空输入框和预览状态
5. 更新每日摘要通知

#### 3. `createSubtasksFromPreview(subtasks)`
**功能**：在预览模式中存储子任务（不保存到数据库）

**逻辑**：
1. 将子任务存储到`previewTaskData.subtasks`
2. 等待用户点击"确认保存"后再创建

## 📊 数据流

### **快速输入模式数据流**

```
全屏编辑输入
    ↓
closeFullscreenDesc()
    ↓
保存到临时变量（tempDescription）
    ↓
返回首页
    ↓
用户调整属性
    ↓
点击"✓"提交
    ↓
addTask()
    ↓
任务创建完成
```

### **预览保存模式数据流**

```
全屏编辑输入
    ↓
previewTask()
    ↓
创建预览任务对象（previewTaskData）
    ↓
打开预览弹窗（TaskDetailModal）
    ↓
用户编辑/AI拆分
    ↓
子任务存储到previewTaskData.subtasks
    ↓
点击"✅ 确认保存"
    ↓
saveTaskFromPreview()
    ↓
创建父任务 + 所有子任务（原子性）
    ↓
任务创建完成
```

## 🐛 常见问题

### **Q1: 点击"完成"后任务没有保存？**
A: "完成"按钮只是关闭全屏编辑，不会保存任务。需要返回首页后点击"✓"提交按钮才能创建任务。

### **Q2: 预览弹窗中的子任务什么时候保存？**
A: 子任务只有在点击"✅ 确认保存"后才会保存到数据库。在此之前，子任务只是存储在内存中。

### **Q3: 预览模式下可以编辑任务吗？**
A: 可以。预览弹窗中可以编辑任务的所有属性，包括标题、描述、分类、优先级等。

### **Q4: 预览模式下可以删除任务吗？**
A: 不可以。预览模式下"删除任务"按钮会被隐藏，因为任务还没有创建。

### **Q5: 预览模式下可以标记完成吗？**
A: 不可以。预览模式下"标记完成"按钮会被隐藏，因为任务还没有创建。
