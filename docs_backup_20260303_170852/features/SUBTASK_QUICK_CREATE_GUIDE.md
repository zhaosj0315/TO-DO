# 🌳 子任务智能识别功能说明

**版本**: v0.7.9  
**实施日期**: 2026-03-02  
**功能类型**: 智能提示功能

---

## 📋 功能概述

子任务智能识别功能提供**自动列表识别**，当用户在任务描述中输入列表时，系统会智能检测并提示用户使用AI拆分功能或预览功能进行更完善的子任务创建。

这是一个**轻量级的智能提示功能**，引导用户使用现有的**AI智能拆分**或**任务预览**功能。

---

## 🎯 功能定位

### **任务分解引导系统**

```
用户输入列表 → 自动识别 → 智能提示 → 引导使用AI拆分或预览功能
```

**与其他功能的关系**：
- 本功能：智能检测 + 友好提示
- AI拆分：完整的子任务创建功能（分类、优先级、日期、依赖关系）
- 任务预览：预览、编辑、AI拆分后一次性保存

---

## 🚀 功能详解

### **自动识别列表**

**工作原理**：
当用户在全屏编辑模式输入任务描述时，系统会自动检测以下列表格式：

```
支持的列表格式：
1. 数字列表：1. 任务名 或 1、任务名
2. 破折号列表：- 任务名
3. 星号列表：* 任务名
4. 圆点列表：• 任务名 或 · 任务名
5. 圆圈数字：① 任务名
```

### **触发条件**

- 描述中至少包含**2个**列表项
- 每个列表项长度在1-100字之间
- 关闭全屏编辑时自动检测

### **用户体验**

1. 用户输入任务描述（包含列表）
2. 点击"完成"关闭全屏编辑
3. 系统弹出提示气泡：
   ```
   🌳 检测到 X 个子任务，建议使用AI拆分功能
   
   预览：
   1. 第一个子任务
   2. 第二个子任务
   3. 第三个子任务
   +X 个子任务
   
   💡 提示：创建任务后，在任务详情页点击"🤖 AI拆分"可以智能拆分子任务
   
   [✕ 知道了]
   ```

### **示例**

**输入描述**：
```
完成项目报告
1. 收集数据
2. 分析结果
3. 撰写报告
4. 提交审核
```

**系统响应**：
- 检测到4个子任务
- 弹出提示气泡
- 引导用户使用AI拆分功能

## 🛠️ 功能2：手动添加子任务


### **操作流程**

1. **打开子任务管理界面**
   - 点击"🌳 子任务"按钮
   - 从底部滑出子任务管理面板

2. **添加子任务**
   - 输入子任务名称（最多100字）

3. **管理子任务**
   - 查看已添加的子任务列表
   - 点击"✕"删除单个子任务
   - 点击"🗑️ 清空全部"删除所有子任务

4. **提交任务**
   - 关闭全屏编辑
   - 点击主任务的"✓"提交按钮
   - 系统自动创建主任务和所有子任务

## 🔗 依赖关系自动设置

### **依赖链规则**

创建子任务时，系统自动建立依赖关系：

```
主任务（父任务）
  ↓ 依赖
子任务1
  ↓ 依赖
子任务2
  ↓ 依赖
子任务3
  ↓ 依赖
子任务4
```

### **命名规则**

- 子任务自动添加序号：`1. 子任务名`
- 子任务描述自动生成：`子任务 1/4（父任务：完成项目报告）`

### **继承属性**

子任务自动继承主任务的以下属性：
- ✅ 任务类型（今天/明天/本周等）
- ✅ 分类（工作/学习/生活）
- ✅ 优先级（高/中/低）
- ✅ 日期时间（指定日期/每周重复等）

## 🔧 技术实现

### **核心函数**

1. **detectSubtasksFromDescription(description)**
   - 功能：从描述中检测列表项
   - 支持5种列表格式
   - 自动去重
   - 返回子任务数组

2. **acceptDetectedSubtasks()**
   - 功能：采纳自动识别的子任务
   - 转换为手动子任务格式
   - 继承主任务属性

3. **toggleManualSubtasks()**
   - 功能：切换手动子任务界面
   - 自动填充检测到的子任务

4. **addManualSubtask()**
   - 功能：添加单个子任务
   - 验证输入（非空、长度限制）
   - 继承主任务属性

5. **addTask()** (修改)
   - 功能：创建主任务和子任务
   - 自动设置依赖关系
   - 生成子任务序号和描述

### **数据结构**

```javascript
// 响应式变量
const showSubtaskSuggestion = ref(false)  // 显示建议气泡

const newSubtaskInput = ref('')           // 输入框内容

{
  text: '子任务名称',
```
