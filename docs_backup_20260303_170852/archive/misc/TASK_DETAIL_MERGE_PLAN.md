# 任务详情页面融合方案

## 📋 现状分析

### 页面1：TaskDetailModal（详情页）
**优点**：
- ✅ 执行日志展示（完整）
- ✅ 番茄钟历史
- ✅ 执行统计
- ✅ 时间轴可视化
- ✅ AI摘要功能

**缺点**：
- ❌ 不可编辑
- ❌ 需要点击"编辑"跳转

### 页面2：编辑弹窗（Bottom Sheet）
**优点**：
- ✅ 可编辑所有字段
- ✅ 分组清晰
- ✅ 字符计数

**缺点**：
- ❌ 功能单一
- ❌ 无日志展示

## 🎯 融合方案

### 设计思路
将TaskDetailModal改为**可编辑模式**，点击字段直接编辑

### 融合后的页面结构
```
┌─────────────────────────────────┐
│ 📋 任务标题 [可编辑]        ✏️ │
├─────────────────────────────────┤
│ 📝 描述 [可编辑]                │
├─────────────────────────────────┤
│ 任务属性 [可编辑]               │
│ • 状态 • 分类 • 优先级          │
│ • 类型 • 日期                   │
├─────────────────────────────────┤
│ 执行统计 [只读]                 │
│ • 推进次数 • 总耗时             │
├─────────────────────────────────┤
│ 时间轴 [只读]                   │
├─────────────────────────────────┤
│ 执行日志 [可添加]               │
│ • 日志列表                      │
│ • [+ 添加日志] 按钮             │
├─────────────────────────────────┤
│ 番茄钟历史 [只读]               │
├─────────────────────────────────┤
│ AI摘要 [可生成]                 │
├─────────────────────────────────┤
│ [完成任务] [删除任务] [关闭]   │
└─────────────────────────────────┘
```

## 🔧 实现步骤

### 1. 保留TaskDetailModal为主体
- 保留所有展示功能
- 添加编辑能力

### 2. 字段改为可编辑
- 标题：点击变为输入框
- 描述：点击变为文本域
- 属性：点击变为选择器

### 3. 删除独立编辑弹窗
- 移除editingTask相关代码
- 移除Bottom Sheet编辑弹窗

### 4. 统一操作按钮
- 底部：[完成任务] [删除任务] [关闭]
- 顶部：[✏️编辑模式切换]

## 💡 交互设计

### 编辑模式切换
- 默认：查看模式（只读）
- 点击✏️：编辑模式（可编辑）
- 编辑模式下：
  - 标题/描述可点击编辑
  - 属性显示为选择器
  - 显示[保存]按钮

### 保存逻辑
- 实时保存（失焦自动保存）
- 或点击[保存]按钮统一保存

## ✅ 优势

1. **简化流程**：一个页面完成所有操作
2. **保留功能**：所有功能都保留
3. **提升体验**：减少页面跳转
4. **统一风格**：界面更一致
