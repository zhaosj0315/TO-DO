### 2️⃣ v1.5.9 (2026-02-21 01:17)
**提交**: `4291b2d`  
**标题**: 修复待办统计与筛选不一致问题，优化弹窗留白

#### 核心Bug修复
**问题**: 待办统计显示1个，但列表显示多个任务

**原因**: 
```javascript
// offlineTaskStore.js 修复前
if (statusFilter === 'pending') {
  filtered = filtered.filter(t => t.status !== 'completed')  // 错误：包含overdue
}
```

**修复**:
```javascript
// 修复后
if (statusFilter === 'pending') {
  filtered = filtered.filter(t => t.status === 'pending')  // 正确：只显示pending
}
```

**影响**: 解决了待办统计与列表显示不一致的严重bug

**标题**: 优化输入框占位符文本，区分搜索和新建功能

#### 问题
两个输入框功能不同但外观相似，用户容易混淆

#### 修改
**修改前**:
- 搜索框：`🔍 搜索任务名称或描述...`
- 新建框：`输入任务名称...` ❌ 不够明确

- 新建框：`➕ 新建任务：输入任务名称...` ✅ 明确是新建

#### 效果
用户一眼就能看出：
- 🔍 = 搜索已有任务
- ➕ = 新建任务
