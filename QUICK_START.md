## 🎨 主要特性

### 1. 任务详情页面
- 全屏弹窗，紫色渐变主题
- 任务概览 + 执行统计 + 日志列表
- 点击任务卡片即可打开

### 2. 添加日志
- 6种类型：开始/进展/阻碍/方案/里程碑/完成
- 必填：类型 + 内容
- 可选：耗时、进度、标签、心情、评分、经验教训
- 时间自动记录

### 3. 执行统计
- 推进次数
- 总耗时 / 平均耗时
- 阻碍统计（总数 / 已解决）
- 进度历史
- 标签聚合

### 4. 任务卡片
- 💬 日志数量徽章
- 📊 进度徽章
- 点击打开详情

## 🔍 查看数据

### 浏览器开发者工具
1. 打开浏览器开发者工具（F12）
2. Application → Storage → Preferences
3. 找到 `tasks_<username>` 键
4. 查看任务数据（包含 logs 和 stats）

### 控制台
```javascript
// 在浏览器控制台执行
const { Preferences } = await import('@capacitor/preferences')
const { value } = await Preferences.get({ key: 'tasks_<username>' })
console.log(JSON.parse(value))
```

## 📝 使用建议

### 场景1：日常任务
- 简单任务可以不添加日志，直接完成
- 复杂任务建议添加日志记录过程

### 场景2：项目任务
1. 开始时添加"开始"日志
2. 每次推进添加"进展"日志，更新进度
3. 遇到问题添加"阻碍"日志
4. 解决问题添加"方案"日志，关联阻碍
5. 重要节点添加"里程碑"日志
6. 完成时添加"完成"日志，总结经验

### 场景3：长期项目
- 使用标签组织日志（如：#需求分析 #开发 #测试）
- 记录心情，了解工作状态
- 完成时评分，积累经验教训

## 🎓 最佳实践

1. **及时记录**：每次推进后立即添加日志
2. **详细描述**：内容尽量详细，便于回顾
3. **使用标签**：用标签分类日志，便于查找
4. **记录阻碍**：遇到问题及时记录，方便总结
5. **完成总结**：任务完成时认真总结，积累经验
