## 🔍 发现的问题

### 问题1：没有任务选择下拉框 ⚠️
**现状**: 
- 代码中有 `centerTaskId` prop，但没有UI让用户选择
- 用户无法指定"查看某个任务的关系"

**影响**: 
- 功能不完整
- 用户体验受限

### 问题2：移动端体验待优化 ⚠️
**现状**:
- 控制按钮在小屏上拥挤（4个按钮+搜索框）
- 图例和统计信息占用空间
- 节点拖动在触摸屏上不够流畅

**影响**:
- 移动端操作不便
- 信息密度过高

### 问题3：性能优化空间 ⚠️
**现状**:
- 显示所有未完成任务（可能上百个）
- 没有分页或虚拟化
- 大量节点时渲染慢

**影响**:
- 任务多时卡顿
- 图谱混乱

### 问题4：关系可读性 ⚠️
**现状**:
- 三种关系线条颜色不同，但没有图例说明
- 用户不知道紫色/红色/绿色分别代表什么

**影响**:
- 学习成本高
- 容易混淆

### 优先级4：性能优化

#### 需求
优化大量任务时的渲染性能

#### 实现方案
1. **节点数量限制**：
   - 全部任务模式：最多显示50个节点
   - 按关系数量排序，优先显示关系多的任务

2. **懒加载**：
   - 初始只显示核心节点
   - 点击节点展开其关系

3. **虚拟化**：
   - 使用 ECharts 的 dataZoom 组件
   - 支持缩放和平移

```javascript
// 限制节点数量
const limitedTasks = computed(() => {
  if (selectedTaskId.value) {
    return getRelatedTasks(selectedTaskId.value)
  } else {
    // 全部任务模式：只显示关系最多的50个
    return taskStore.tasks
      .filter(t => t.status !== 'completed')
      .map(t => ({
        ...t,
        relationCount: (t.linkedTasks?.length || 0) + 
                      (t.waitFor?.length || 0) + 
                      (t.subtasks?.length || 0)
      }))
      .sort((a, b) => b.relationCount - a.relationCount)
      .slice(0, 50)
  }
})
```
