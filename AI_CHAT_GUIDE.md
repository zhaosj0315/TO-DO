# AI问答功能使用指南

## 功能概述

AI问答功能让你可以通过自然语言与大模型对话，智能分析你的任务数据。所有任务信息（包括标题、描述、状态、日志、番茄钟等）都会作为上下文提供给AI。

## 使用方法

### 1. 打开AI问答
点击页面右上角的 🤖 图标（位于演示模式按钮左侧）

### 2. 配置模型
首次使用需要选择并配置模型：

#### 盘古大模型（推荐）
- 选择"盘古大模型"
- 输入API地址，例如：`http://localhost:8000/api/chat`
- 如果使用远程服务，填写完整URL

#### OpenAI
- 选择"OpenAI"
- 输入你的API Key
- 使用gpt-3.5-turbo模型

#### 本地模型
- 选择"本地模型"
- 输入Ollama地址，默认：`http://localhost:11434`
- 需要本地安装Ollama并运行llama2模型

### 3. 开始提问

#### 快捷问题（推荐）
首次打开会显示6个常用问题按钮：
- 今天完成了什么？
- 本周完成情况
- 高优先级待办
- 效率分析
- 阻碍统计
- 管理建议

#### 自由提问
你可以问任何关于任务的问题，例如：

**时间维度**
- "今天我做了什么？"
- "本周完成了哪些任务？"
- "本月的任务完成率是多少？"

**状态查询**
- "有哪些待办任务？"
- "哪些任务已经逾期了？"
- "我完成了多少个高优先级任务？"

**分类统计**
- "工作类任务有多少？"
- "学习任务的完成情况如何？"
- "生活类任务占比多少？"

**执行分析**
- "我遇到了哪些阻碍？"
- "哪些任务有执行日志？"
- "我的任务执行效率怎么样？"

**番茄钟统计**
- "今天完成了几个番茄钟？"
- "哪个任务的番茄钟最多？"
- "我的专注时长统计"

**建议咨询**
- "给我一些任务管理建议"
- "如何提高完成率？"
- "我应该优先做哪些任务？"

## 数据说明

AI可以访问的数据包括：

### 基础信息
- 任务标题、描述
- 创建时间、完成时间
- 状态（待办/已完成/已逾期）
- 分类（工作/学习/生活）
- 优先级（高/中/低）

### 执行数据
- 执行日志（开始/进展/阻碍/方案/里程碑/完成）
- 进度百分比
- 耗时统计
- 阻碍和解决方案

### 番茄钟数据
- 已完成番茄钟数量
- 番茄钟历史记录
- 专注时长统计

### 统计数据
- 今日/本周/本月创建和完成数量
- 完成率
- 回收站任务

## 技术细节

### 数据格式
所有数据以Markdown格式提供给AI，包含：
- 📊 整体统计
- 📅 今日数据
- 📆 本周数据
- 📈 本月数据
- 📋 所有任务详情
- 🗑️ 回收站任务

### API接口

#### 盘古大模型
```json
POST {panguUrl}
{
  "messages": [
    { "role": "system", "content": "完整的任务数据上下文" },
    { "role": "user", "content": "用户问题" }
  ],
  "stream": false
}
```

#### OpenAI
```json
POST https://api.openai.com/v1/chat/completions
{
  "model": "gpt-3.5-turbo",
  "messages": [...]
}
```

#### 本地模型（Ollama）
```json
POST {localUrl}/api/generate
{
  "model": "llama2",
  "prompt": "上下文 + 问题",
  "stream": false
}
```

## 注意事项

1. **数据隐私**：所有数据仅在对话时发送给AI，不会永久存储
2. **网络要求**：需要能访问对应的API服务
3. **配置保存**：API地址和Key会保存在浏览器localStorage
4. **上下文限制**：如果任务数量过多，可能超出模型上下文长度限制
5. **响应时间**：根据模型和网络情况，响应时间可能需要几秒到几十秒

## 常见问题

**Q: 为什么AI回答不准确？**
A: 确保选择了正确的模型，并且API配置正确。盘古大模型需要正确的API地址。

**Q: 可以问哪些问题？**
A: 任何关于你任务数据的问题都可以，AI会基于实际数据回答。

**Q: 数据会被保存吗？**
A: 不会。数据仅在对话时临时发送给AI，对话结束后不会保留。

**Q: 支持连续对话吗？**
A: 是的，每次对话都会包含完整的任务数据上下文。

**Q: 如何切换模型？**
A: 在对话窗口顶部的下拉菜单中选择不同的模型类型。
