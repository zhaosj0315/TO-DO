# 报告日期计算修复说明

## 问题描述
用户反馈周报显示"2026/3/8 至 2026/3/14"（未来一周），但实际应该显示"本周一到今天"的数据。

## 根本原因

### 1. 周报日期计算错误
**修复前**：
```javascript
case 'weekly':
  startDate = new Date(now)
  startDate.setDate(now.getDate() - now.getDay())  // ❌ 周日时getDay()=0，变成上周日
  endDate = new Date(startDate)
  endDate.setDate(startDate.getDate() + 6)  // ❌ 未来的周日
```

**问题**：
- `now.getDay()` 在周日返回0，导致 `startDate` 变成**上周日**
- `endDate` 设置为未来的周日，而非今天

**修复后**：
```javascript
case 'weekly':
  const dayOfWeek = now.getDay()
  const daysFromMonday = dayOfWeek === 0 ? 6 : dayOfWeek - 1  // ✅ 周日回退6天
  
  startDate = new Date(now)
  startDate.setDate(now.getDate() - daysFromMonday)  // ✅ 本周一
  startDate.setHours(0, 0, 0, 0)
  
  endDate = new Date(now)  // ✅ 今天
  endDate.setHours(23, 59, 59, 999)
```

### 2. 其他报告类型也有相同问题

| 报告类型 | 修复前 endDate | 修复后 endDate |
|---------|---------------|---------------|
| 月报 | 本月最后一天 | 今天 |
| 季报 | 本季度最后一天 | 今天 |
| 半年报 | 本半年最后一天 | 今天 |
| 年报 | 12月31日 | 今天 |

**原因**：报告应该总结"截止到今天已完成的工作"，而非"整个周期的计划"。

## 总结

1. **核心问题**：所有报告类型的 `endDate` 都设置为周期结束日期，而非今天
2. **设计理念**：报告是"总结过去"而非"规划未来"
3. **修复策略**：统一将 `endDate` 改为 `new Date(now)`
4. **特殊处理**：周报需要额外处理周日的边界情况
