## 🎯 测试策略

### 测试金字塔

```
        /\
       /  \  E2E测试 (10%)
      /────\
     /      \  集成测试 (20%)
    /────────\
   /          \  单元测试 (70%)
  /────────────\
```

### 测试原则

1. **离线优先**: 所有测试必须在离线环境下通过
2. **数据隔离**: 测试数据不影响生产数据
3. **可重复性**: 测试结果可重复
4. **自动化**: 核心功能自动化测试
5. **真实场景**: 模拟真实用户操作

## 🛠️ 测试环境

### 开发环境

**浏览器测试**:
```bash
npm run dev
# 访问 http://localhost:5173
```

**Android 测试**:
```bash
npx cap sync android
npx cap open android
# 在 Android Studio 中运行
```

**Windows 测试**:
```bash
npm run build
npm run electron:build
# 运行生成的安装包
```

### 测试数据

**导入测试数据**:
1. 下载 `TODO导入模板示例.xlsx`
2. 登录测试账号
3. 点击"导入"按钮
4. 选择模板文件
5. 确认导入 100 条测试任务

**清空测试数据**:
```bash
node scripts/清理本地数据.js
```
